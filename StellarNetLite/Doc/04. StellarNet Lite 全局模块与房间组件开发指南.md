# 04. StellarNet Lite 全局模块与房间组件开发指南

> 本文讲最常用的开发入口：  
> **什么时候写全局模块，什么时候写房间组件。**

如果你已经理解这篇里的边界划分，想直接照着做一个完整业务链路，继续看：

- `Doc/03. StellarNet Lite 30分钟业务开发上手实践.md`

---

## 1. 先判断你的功能属于哪一层

### 1.1 属于大厅或全局状态

这类功能写**全局模块**。

典型例子：

- 登录
- 拉房间列表
- 大厅聊天
- 公告
- 录像列表和下载

### 1.2 属于房间内部状态

这类功能写**房间组件**。

典型例子：

- 准备
- 开始游戏
- 对象同步
- 房间内移动
- 房间内聊天或表情

---

## 2. 当前代码里的两种组织方式

### 2.1 全局模块

服务端：

- `[ServerModule(...)]`

客户端：

- `[ClientModule(...)]`

它们处理的是 `NetScope.Global` 协议。

### 2.2 房间组件

服务端和客户端都用：

- `[RoomComponent(...)]`

它们处理的是 `NetScope.Room` 协议。

---

## 3. 开发一个功能的标准文件分布

不管你写模块还是写组件，最常见的代码分布都是三块：

1. `Shared/Protocol`
2. `Server`
3. `Client`

也就是：

- 协议定义在 Shared
- 权威处理在 Server
- 接收与表现驱动在 Client

---

## 4. 先写 Shared：定义协议

协议定义必须放在 Shared。

例如做一个全局公告请求：

```csharp
using StellarNet.Lite.Shared.Core;

[NetMsg(8201, NetScope.Global, NetDir.C2S)]
public sealed class C2S_RequestAnnouncement
{
    public string Content;
}

[NetMsg(8202, NetScope.Global, NetDir.S2C)]
public sealed class S2C_AnnouncementSync
{
    public string SenderName;
    public string Content;
}
```

这里最重要的是三个信息：

- `MsgId`
- `Scope`
- `Dir`

写错任意一个，后面路由都会错。

---

## 5. 再写 Server：处理真实业务

### 5.1 全局模块示例

```csharp
using StellarNet.Lite.Server.Core;
using StellarNet.Lite.Shared.Protocol;

[ServerModule("ServerAnnouncementModule", "公告模块")]
public sealed class ServerAnnouncementModule
{
    private readonly ServerApp _app;

    public ServerAnnouncementModule(ServerApp app)
    {
        _app = app;
    }

    [NetHandler]
    public void OnC2S_RequestAnnouncement(Session session, C2S_RequestAnnouncement msg)
    {
        if (session == null || msg == null || string.IsNullOrWhiteSpace(msg.Content))
        {
            return;
        }

        string senderName = string.IsNullOrEmpty(session.AccountId)
            ? session.SessionId
            : session.AccountId;

        var syncMsg = new S2C_AnnouncementSync
        {
            SenderName = senderName,
            Content = msg.Content.Trim()
        };

        foreach (var kvp in _app.Sessions)
        {
            Session target = kvp.Value;
            if (target != null && target.IsOnline)
            {
                _app.SendMessageToSession(target, syncMsg);
            }
        }
    }
}
```

这个例子说明了全局模块最常见的职责：

- 校验请求
- 读取服务端事实状态
- 组装结果包
- 发给目标客户端

### 5.2 房间组件示例

```csharp
using StellarNet.Lite.Server.Core;

[RoomComponent(930, "RoomNotice", "房间提示组件")]
public sealed class ServerRoomNoticeComponent : ServerRoomComponent
{
    [NetHandler]
    public void OnC2S_SendRoomNotice(Session session, C2S_SendRoomNotice msg)
    {
        if (session == null || msg == null || string.IsNullOrWhiteSpace(msg.Content))
        {
            return;
        }

        Room.BroadcastMessage(new S2C_RoomNoticeSync
        {
            SenderSessionId = session.SessionId,
            Content = msg.Content.Trim()
        });
    }
}
```

房间组件和全局模块最大的区别不是写法，而是**上下文**：

- 全局模块通过 `ServerApp` 看全局
- 房间组件通过 `Room` 看当前房间

如果服务端房间组件需要访问当前服务端宿主上的全局模块实例，当前也可以通过组件里持有的 `_app` 调用：

- `_app.GetGlobalModule<TModule>()`
- `_app.TryGetGlobalModule<TModule>(out TModule module)`

而房间内的其他房间组件，仍然通过：

- `Room.GetComponent<TOtherComponent>()`

## 6. 再写 Client：接收结果并抛给表现层

### 6.1 全局模块的客户端写法

```csharp
using StellarNet.Lite.Client.Core;
using StellarNet.Lite.Client.Core.Events;

[ClientModule("ClientAnnouncementModule", "公告模块")]
public sealed class ClientAnnouncementModule
{
    [NetHandler]
    public void OnS2C_AnnouncementSync(S2C_AnnouncementSync msg)
    {
        if (msg == null) return;
        GlobalTypeNetEvent.Broadcast(msg);
    }
}
```

### 6.2 房间组件的客户端写法

```csharp
using StellarNet.Lite.Client.Core;

[RoomComponent(930, "RoomNotice", "房间提示组件")]
public sealed class ClientRoomNoticeComponent : ClientRoomComponent
{
    [NetHandler]
    public void OnS2C_RoomNoticeSync(S2C_RoomNoticeSync msg)
    {
        if (msg == null || Room == null) return;
        Room.NetEventSystem.Broadcast(msg);
    }
}
```

这里的关键设计是：

- 客户端组件和模块通常不直接操作 UI
- 更推荐先抛事件，再由表现层监听

---

## 7. `GlobalTypeNetEvent` 和 `Room.NetEventSystem` 怎么选

### 7.1 `GlobalTypeNetEvent`

适合：

- 登录结果
- 公告
- 下载进度
- 大厅列表刷新

也就是不依附某个房间生命周期的事件。

### 7.2 `Room.NetEventSystem`

适合：

- 当前房间成员变化
- 房间内对象生成销毁
- 房间内业务通知

也就是必须跟房间实例同生共死的事件。

---

## 8. 房间组件还有哪些生命周期钩子

服务端房间组件基类 `ServerRoomComponent` 当前提供了这些钩子：

- `OnInit()`
- `OnDestroy()`
- `OnMemberJoined(Session session)`
- `OnMemberLeft(Session session)`
- `OnMemberOffline(Session session)`
- `OnMemberOnline(Session session)`
- `OnSendSnapshot(Session session)`
- `OnGameStart()`
- `OnGameEnd()`

所以房间组件不只是“收消息处理器”，也可以是：

- 状态初始化器
- 成员进出房监听器
- 快照导出器
- 对局生命周期响应器

---

## 9. 什么时候要实现 `ITickableComponent`

如果你的组件不是“收到消息才工作”，而是要**按固定频率推进**，就实现：

```csharp
ITickableComponent
```

典型场景：

- AI
- 定时器
- 圈缩
- 周期广播
- 高频状态同步

默认 `ObjectSync` 就是这样工作的。

## 10. 推荐开发流程

当前最顺手的流程是：

1. 用脚手架工具生成基础文件
2. 在 `Shared` 写协议
3. 在 `Server` 写权威逻辑
4. 在 `Client` 写接收逻辑
5. 重新生成协议和组件常量表
6. 进 Unity 运行验证

补充：

- 如果你的功能会在“刚进房但尚未最终确认”的窗口期触发，普通业务代码不用自己额外加 room-scope 发送门。
- 当前 `ClientApp.SendMessage(...)` 会在 `S2C_RoomSetupResult.Success == true` 之前自动拦截 `NetScope.Room` 发包。
- 这让业务层可以在 `S2C_RoomSetupResult.Success == true` 后再做表现层装配，同时又不会把半初始化房间消息送进服务端。

对应工具见：

- `Doc/05. StellarNet Lite 代码生成与编辑器工具链指南.md`

## 11. 初学者最容易写错的地方

### 11.1 把协议写在 Client 或 Server

不推荐。

协议必须放 Shared，否则双端事实会分裂。

### 11.2 在 Client 里直接改权威状态

不推荐。

客户端更适合：

- 缓存
- 表现
- 事件派发

### 11.3 把大厅功能写成房间组件

会导致：

- 不在房间时功能不可用
- 生命周期不对

### 11.4 忘了重新生成自动装配代码

会导致：

- 新协议收不到
- 新组件挂不上
- 新模块不生效

补充：

- 现在新增房间确认闭环后，如果你新增了 `NetMsg` 协议，例如类似 `S2C_RoomSetupResult` 这种补链协议，也同样必须重新生成。
- 否则 `MsgIdConst`、自动绑定器、自动注册表都可能还是旧版本。

### 11.5 以为全局模块默认就能登录前访问

当前不是这样。

- 未鉴权会话不能访问 `Room` 协议。
- 未鉴权会话只能访问显式放行的 `Global` 协议。
- 白名单来源于 `NetMsg(..., AllowUnauthenticated = true)`，默认 `C2S_Login` 已放行。

如果你的业务需要支持登录前全局协议，例如 `C2S_MyLogin`、`C2S_AdvancedLogin` 这类自定义登录入口，不需要再去 Runtime 手工注册白名单；只要满足：

- 协议是 `NetScope.Global`
- 协议是 `NetDir.C2S`
- 在 `[NetMsg(...)]` 上声明 `AllowUnauthenticated = true`
- 重新运行一次 `LiteProtocolScanner`

否则即便模块和处理函数已经写好，也会在 `ServerApp` 的未鉴权门禁处被提前拦截。

---

## 12. 一句话判断规则

如果你还不确定该写模块还是组件，先用这句判断：

- 不依赖房间上下文，用模块
- 必须依附当前房间，用组件
