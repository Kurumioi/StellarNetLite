# 15. StellarNet Lite Runtime 源码逐层参考

本文档专门覆盖 `StellarNetLite/Runtime`。

阅读目标不是背 API 名字，而是搞清楚三件事：

1. 运行时主链从哪里进入、在哪里切分客户端和服务端。
2. 每个核心类持有哪些状态、这些状态为什么存在。
3. 某个问题应该去哪个脚本里排查，而不是盲目全文搜索。

---

## 1. Runtime 总体结构

`Runtime` 可以按职责拆成六块：

- `StellarNetAppManager.cs`
  统一入口，负责装配传输、序列化、配置、自动注册表、运行时扩展桥。
- `Client/Core`
  客户端状态机、房间对象、房间组件装配、全局/房间事件、客户端网络门面。
- `Server/Core`
  服务端状态机、会话、房间、快照、分发器、房间组件装配。
- `Server/Infrastructure`
  服务端线程宿主、房间分片调度、异步持久化、统一出站队列、无头控制台。
- `Shared/Core` 与 `Shared/Infrastructure`
  协议特性、包结构、协议索引、传输接口、配置、日志、主线程桥接、序列化。
- `Transports`
  `TCP` / `KCP` / `UDP` 及包编解码器。

## 2. 运行时入口层

### 2.1 `Runtime/StellarNetAppManager.cs`

文件职责：

- 整个框架的统一运行时入口。
- 在 `Awake` 中安装 `UnityPlayerLoopDispatcher`、装配传输层、序列化器与配置。
- 负责把自动生成绑定器接入 `ServerRoomFactory` / `ClientRoomFactory`。
- 负责创建 `ClientApp`、`ServerApp` 与 `ServerRuntimeHost`。
- 负责把传输层事件桥接到客户端/服务端逻辑宿主。
- 负责重连协程和运行时扩展桥生命周期。

关键关联：

- 上游依赖 `INetworkTransport`、`NetConfigLoader`、`NetMessageMapper`。
- 下游创建 `ClientApp`、`ServerApp`、`ServerRuntimeHost`。
- 通过 `GeneratedRuntimeAccess` 和 `IRuntimeFeatureBridge` 让扩展层接入而不反向污染 Runtime。

重要字段/属性：

- `Serializer`
  当前序列化器，默认是 `LiteNetSerializer`。
- `Transport`
  当前场景上挂载的物理传输组件。
- `_netConfig`
  当前运行时配置快照。
- `_serverRuntimeRequested`
  是否请求维持服务端运行，用来区分“已停服”和“传输层尚未回调”的时序。
- `_serverLifecycleLock`
  服务端宿主创建/销毁互斥锁，避免重复启动或并发回收。
- `_serverRuntime`
  真实的服务端后台宿主线程。
- `_runtimeFeatureBridges`
  当前所有运行时扩展桥实例。
- `ServerApp`
  当前服务端逻辑宿主。
- `ClientApp`
  当前客户端逻辑宿主。
- `_reconnectCoroutine`
  自动重连协程句柄。

关键函数：

- `Awake()`
  安装主线程桥接器，装配 transport/serializer/config，初始化消息映射器和自动绑定器，注册传输事件。
- `ApplyConfig(NetConfig config)`
  热更新当前运行时配置并同步给传输层。
- `StartClient()/StopClient()/StartServer()/StopServer()/StartHost()`
  对外暴露的物理层快捷代理。
- `EnsureServerRuntimeCreated()`
  真正创建 `ServerApp`、注册未鉴权协议、装配自动模块、创建 `ServerRuntimeHost`。
- `DisposeServerRuntime()`
  停止宿主线程、销毁 `ServerApp`、清空房间工厂注册。
- `HandleServerStarted/Stopped/ClientConnected/...`
  负责把传输层物理事件翻译成 Runtime 层状态变更。
- `HandleServerReceivePacket`
  将服务端收包转交给 `ServerRuntimeHost` 线程。
- `HandleClientReceivePacket`
  客户端收包时先给扩展桥，再交给 `ClientApp`。
- `ReconnectionRoutine()`
  在线房间断线后的 15 秒自动重试循环；期间广播本地事件并定时 `Transport.StartClient()`。
- `RestartReconnectionRoutine()`
  允许扩展或 UI 在挂起态下重置重连窗口。

设计重点：

- `StellarNetAppManager` 不直接处理业务协议，它只负责装配和转发。
- 所有扩展逻辑通过 `IRuntimeFeatureBridge` 插入。
- 服务端逻辑严格收敛到 `ServerRuntimeHost` 线程。

### 2.2 `Runtime/GeneratedRuntimeAccess.cs`

文件职责：

- 作为 Runtime 对自动生成代码的“安全访问层”。
- 当某些生成文件暂时不存在时，避免 Runtime 编译直接炸掉。

类型与成员：

- `GeneratedRuntimeAccess`
  - `RuntimeFeatureRegistryTypeName`
    生成的运行时扩展桥注册表完整类型名。
  - `UnauthenticatedRegistryTypeName`
    未鉴权协议白名单注册表完整类型名。
  - `CreateRuntimeFeatureBridges()`
    反射调用生成注册表的 `CreateFeatures`。
  - `GetUnauthenticatedGlobalC2SMsgIds()`
    反射读取白名单字段 `GlobalC2SMsgIds`。
  - `ResolveType(string fullName)`
    在当前程序集内查找生成类型。

### 2.3 `Runtime/IRuntimeFeatureBridge.cs`

文件职责：

- 定义扩展层接入 Runtime 的统一桥接口。
- Runtime 只认接口，不认扩展具体类型。

类型与成员：

- `IRuntimeFeatureBridge`
  - `OnRuntimeAwake`
    Runtime 宿主 `Awake` 完成时调用。
  - `OnServerAppCreated`
    `ServerApp` 创建完成时调用。
  - `OnServerSessionStateChanged`
    会话在线/离线/绑定变化时调用。
  - `TryNotifyServerSessionKick`
    服务端踢人前让扩展补发业务通知。
  - `OnClientAppCreated`
    `ClientApp` 创建完成时调用。
  - `OnClientConnected`
    客户端物理连接成功时调用。
  - `OnClientDisconnected`
    客户端物理断开时调用。
  - `OnClientPacketReceived`
    客户端收到网络包后调用。
  - `OnClientStopped`
    客户端停止时调用。
- `RuntimeFeatureBridgeBase`
  所有方法默认空实现，减少扩展样板代码。

## 3. Client/Core

### 3.1 `Runtime/Client/Core/ClientApp.cs`

文件职责：

- 客户端逻辑主状态机。
- 维护当前客户端状态、会话、当前房间、全局分发器。
- 负责所有 C2S 发包前的合法性检查。

类型：

- `ClientAppState`
  - `InLobby`
    大厅态。
  - `OnlineRoom`
    正式在线房间态。
  - `SandboxRoom`
    本地沙盒或纯回放房间态。
  - `ConnectionSuspended`
    物理连接断开、等待恢复态。
- `ClientApp`

关键状态：

- `Session`
  当前客户端会话对象。
- `GlobalDispatcher`
  全局协议分发器。
- `CurrentRoom`
  当前在线房间或回放房间。
- `State`
  当前状态机状态。
- `IsNetworkBlocked`
  弱网阻断时的发包总闸。
- `_sendSeq`
  客户端单会话内的发送序号。
- `_isCurrentRoomConfirmed`
  是否已经拿到 `RoomSetupResult.Success`。
- `_weakNetBypassMsgIds`
  弱网阻断/挂起态仍允许发送的协议白名单。

关键函数：

- `RegisterWeakNetBypassProtocol(int)` / `RegisterWeakNetBypassProtocol<T>()`
  注册弱网豁免发包协议。
- `Dispose()`
  深度销毁当前客户端逻辑和房间。
- `RegisterDisposeCallback(Action)`
  给扩展提供随 `ClientApp` 一起回收的钩子。
- `ConfirmCurrentRoom(string roomId)`
  标记当前在线房间已最终确认。
- `OnReceivePacket(Packet packet)`
  按 `NetScope` 转给全局分发器或房间分发器。
- `EnterOnlineRoom(string roomId)`
  切入正式在线房间并绑定 `ClientSession`。
- `EnterSandboxRoom(string roomId)`
  切入本地沙盒/回放房间。
- `LeaveRoom(bool silent = false)`
  只清理客户端本地房间上下文，不一定意味着已通知服务端。
- `SuspendConnection()`
  在线房间断线时进入软挂起态。
- `AbortConnection()`
  硬中止会话并回大厅态。
- `SendMessage<T>(T msg)`
  所有客户端发包总入口；会做消息元信息、方向、状态、弱网、房间确认等校验。
- `ExecuteDisposeCallbacks()`
  执行扩展注册的销毁回调。

### 3.2 `Runtime/Client/Core/ClientSession.cs`

文件职责：

- 持有客户端本地会话状态，不负责网络发送。

字段/属性：

- `SessionId`
  登录成功后服务端分配的会话标识。
- `AccountId`
  当前登录账号。
- `CurrentRoomId`
  当前正式所在房间。
- `IsLoggedIn`
  通过 `SessionId` 判断是否已登录。
- `IsPhysicalOnline`
  底层连接是否在线。
- `LastBoundRoomId`
  最近一次成功进入的房间。
- `LastDisconnectRealtime`
  最近一次断开时间。
- `IsReconnecting`
  是否正在执行重连链。

函数：

- `SetAccountId`
  只更新账号字段，通常由上层流程用。
- `OnLoginSuccess`
  刷新登录成功后的会话状态。
- `BindRoom`
  绑定当前房间。
- `UnbindRoom`
  清空当前房间。
- `Clear`
  清空整份会话上下文。
- `ClearRecoveryContext`
  仅清理重连恢复上下文。

### 3.3 `Runtime/Client/Core/ClientRoom.cs`

文件职责：

- 客户端房间实例。
- 管理房间组件、房间协议分发器、房间级事件总线。

字段/属性：

- `RoomId`
  当前房间标识。
- `Dispatcher`
  房间域协议分发器。
- `NetEventSystem`
  房间级本地事件系统。
- `_components`
  当前房间已装配的组件列表。
- `Components`
  供回放或视图层按类型查找组件的只读视图。
- `_isDestroyed`
  防止重复销毁。

函数：

- `Create(string roomId)`
  校验并创建房间实例。
- `AddComponent(ClientRoomComponent component)`
  把组件挂到房间上。
- `GetComponent<T>()`
  通过泛型查找组件。
- `InitializeComponents()`
  依次调用各组件 `OnInit`。
- `Destroy()`
  依次调用各组件 `OnDestroy`，清空分发器和事件系统。

### 3.4 `Runtime/Client/Core/ClientRoomComponent.cs`

文件职责：

- 客户端房间组件基类。

成员：

- `Room`
  组件所属房间，由工厂在装配时注入。
- 典型虚函数：
  实际基类里只保留最小宿主关系，业务组件通过继承扩展自己的 `OnInit/OnDestroy` 和 `[NetHandler]`。

### 3.5 `Runtime/Client/Core/ClientRoomFactory.cs`

文件职责：

- 按 `ComponentId` 创建并装配客户端房间组件。

关键成员：

- `ComponentBinder`
  自动生成的消息处理器绑定器。
- `Registry`
  `ComponentId -> 构造器` 映射表。
- `Register`
  注册组件构造器。
- `Clear`
  清空注册表。
- `BuildComponents`
  按组件 ID 批量创建、挂载、绑定消息并最终初始化。

### 3.6 `Runtime/Client/Core/ClientDispatchers.cs`

文件职责：

- 保存客户端全局域与房间域消息处理表。

类型与成员：

- `ClientGlobalDispatcher`
  - `_handlers`
    `MsgId -> Action<Packet>`。
  - `Register`
    注册全局协议处理器。
  - `Dispatch`
    分发一条全局协议。
  - `Clear`
    清空处理器。
- `ClientRoomDispatcher`
  - `_handlers`
    房间域处理器表。
  - `_roomId`
    用于日志定位。
  - `Register/Dispatch/Clear`
    与全局分发器一致，只是带房间语义。

### 3.7 `Runtime/Client/Core/NetClient.cs`

文件职责：

- 客户端静态网络门面。
- 让业务层在不知道 `ClientApp` 实例引用的情况下也能发消息和查状态。

成员：

- `_app`
  当前绑定的 `ClientApp`。
- `App` / `Session` / `State` / `CurrentRoom`
  常用状态快速访问器。
- `Initialize(ClientApp app)`
  绑定或清空当前客户端主状态机。
- `Send<T>(T msg)`
  静态发包入口，内部转到 `ClientApp.SendMessage`。

### 3.8 `Runtime/Client/Core/Events/GlobalTypeNetEvent.cs`

文件职责：

- 全局强类型事件总线。
- 不是网络协议分发器，而是客户端本地事件系统。

关键类型与成员：

- `GlobalTypeNetEvent`
  - `Register<T>`
    注册某类型事件的回调，返回注销令牌。
  - `UnRegister<T>`
    显式注销单个监听。
  - `Broadcast<T>(T e)` / `Broadcast<T>()`
    广播事件实例或默认实例。
  - `UnRegisterAll<T>`
    清空某事件类型的全部监听。
- `EventBox<T>`
  保存某一种事件类型的订阅者与令牌池。
- `EventToken`
  令牌实现，支持：
  - `UnRegister`
  - `UnRegisterWhenGameObjectDestroyed`
  - `UnRegisterWhenMonoDisable`

### 3.9 `Runtime/Client/Core/Events/IUnRegister.cs`

文件职责：

- 定义注销令牌接口与默认触发器实现。

类型与成员：

- `IUnRegister`
  - `UnRegister`
  - `UnRegisterWhenGameObjectDestroyed`
  - `UnRegisterWhenMonoDisable`
- `CustomUnRegister`
  基于 `Action` 的默认令牌实现。
- `EventUnregisterTrigger`
  `GameObject` 销毁时批量注销。
- `EventUnregisterDisableTrigger`
  `MonoBehaviour` 禁用时批量注销。

### 3.10 `Runtime/Client/Core/Events/RoomNetEventSystem.cs`

文件职责：

- 房间内局部事件总线。
- 作用域比 `GlobalTypeNetEvent` 小，只绑定到某个 `ClientRoom`。

成员：

- `_delegates`
  `Type -> Delegate` 映射。
- `_roomId`
  仅用于日志和定位。
- `Register<T>`
  注册房间事件。
- `UnRegister<T>`
  注销房间事件。
- `Broadcast<T>`
  广播房间事件。
- `Clear`
  清空所有房间事件监听。

### 3.11 `Runtime/Client/Core/Events/LocalEvents/LocalNetEvents.cs`

文件职责：

- 定义 Runtime 自己广播的本地事件结构体。

类型与字段：

- `Local_ConnectionSuspended`
  - `RemainingSeconds`
    剩余自动重连秒数。
- `Local_ReconnectTimeout`
  自动重连超时事件，无字段。
- `Local_SystemPrompt`
  - `Message`
    轻提示文本。
- `Local_ConnectionAborted`
  硬中止连接事件，无字段。

## 4. Server/Core

### 4.1 `Runtime/Server/Core/ServerApp.cs`

文件职责：

- 服务端主状态机。
- 持有所有房间、会话、全局模块、未鉴权协议白名单。
- 执行服务端收包、路由、GC、发包入口。

关键状态：

- `GlobalDispatcher`
  全局域消息分发器。
- `PersistenceRuntime`
  异步持久化任务托管器。
- `Rooms` / `Sessions`
  对外暴露的只读房间与会话索引。
- `Config`
  运行时配置。
- `SyncRoot`
  线程同步锁对象。
- `CurrentRealtimeSinceStartup` / `CurrentUtcNow`
  当前服务端逻辑时钟。
- `RoomScheduler`
  房间分片调度器。
- `RoomRecordingService`
  回放录制服务，可由扩展注入。
- `RoomMembershipNotifier`
  强制移出成员通知器，可由扩展注入。
- `_sessions`
  `SessionId -> Session` 主索引。
- `_connectionToSession`
  `ConnectionId -> Session` 物理连接索引。
- `_accountToSession`
  `AccountId -> Session` 账号索引。
- `_rooms`
  `RoomId -> Room`。
- `_globalModules`
  已注册的服务端全局模块实例。
- `_unauthenticatedGlobalMsgIds`
  登录前允许访问的全局协议白名单。

关键函数：

- `UpdateRuntimeContext`
  刷新当前统一时间上下文。
- `AttachRoomScheduler`
  绑定房间调度器。
- `FlushOutboundPackets`
  将 `ServerOutboundDispatcher` 中的出站消息真正推到 transport。
- `CaptureRoomRuntimeSnapshots`
  给大厅、监控和 GC 用的轻量快照接口。
- `CaptureRoomDetailedSnapshot`
  取单房间详情快照。
- `CaptureRoomWorkerStats`
  获取 worker 负载数据。
- `Dispose`
  深度销毁全部房间、会话与模块。
- `Tick`
  执行房间与离线会话 GC。
- `OnReceivePacket`
  服务端收包总入口：匿名会话分配、序号防重放、鉴权校验、按作用域路由。
- `SendMessageToSession<T>`
  服务端向指定会话发送 S2C 协议。
- `CreateRoom/GetRoom/DestroyRoom/RegisterRoomToScheduler`
  房间创建、查询、销毁、接入调度器。
- `RegisterUnauthenticatedGlobalProtocol`
  注册登录前可访问的全局 C2S 协议。
- `RegisterGlobalModule/GetGlobalModule/TryGetGlobalModule`
  管理服务端全局模块。
- `GetSessionByAccountId/RegisterSession/RemoveSession`
  管理会话主索引与派生索引。
- `BindConnection/UnbindConnection`
  处理同一业务会话与物理连接的绑定变化。
- `TryGetSessionByConnectionId`
  底层连接到会话的内部查询。

### 4.2 `Runtime/Server/Core/Session.cs`

文件职责：

- 服务端会话实体。
- 保存连接状态、房间状态、授权状态、序号基线、活跃时间。

关键字段/属性：

- `_connectionId`
  当前物理连接 ID。
- `_currentRoomId`
  当前正式所在房间。
- `_authorizedRoomId`
  当前被授权允许进入的房间。
- `_recoverableRoomId`
  当前唯一可恢复房间。
- `_isOnline`
  物理连接是否在线。
- `_isRoomReady`
  当前成员是否已完成房间装配。
- `_lastOfflineTime`
  最近一次离线时间。
- `_lastReceivedSeq`
  最近一次接收并通过校验的序号。
- `_lastActiveRealtime`
  最近任意业务包活跃时间。
- `_lastRoomActiveRealtime`
  最近房间包活跃时间。
- `SessionId` / `AccountId`
  会话和账号标识。
- `IsAuthenticated`
  是否已经不是 `UNAUTH` 会话。

关键函数：

- `UpdateConnection`
  重绑物理连接并刷新在线态。
- `MarkOffline`
  标记离线并清空房间 ready 状态。
- `MarkActive` / `MarkRoomActive`
  刷新活跃时间。
- `BindRoom` / `UnbindRoom`
  正式绑定或解绑房间。
- `AuthorizeRoom` / `ClearAuthorizedRoom`
  设置或清理进房授权。
- `SetRecoverableRoom` / `ClearRecoverableRoom`
  设置或清理可恢复房间。
- `SetRoomReady`
  更新房间装配完成状态。
- `TryConsumeSeq`
  递增消费消息序号，用于防重放/乱序保护。
- `ResetSeq`
  重置序号基线，常见于顶号或重绑连接。

### 4.3 `Runtime/Server/Core/RoomInstance/Room.cs`

文件职责：

- 服务端房间实体。
- 负责成员管理、组件生命周期、房间广播、录制、重连快照、周期 Tick。

类型：

- `RoomState`
  `Waiting` / `Playing` / `Finished`。
- `Room`

关键状态：

- `RoomId`
  房间 ID。
- `Config`
  `RoomConfigModel`，保存静态房间配置。
- `Dispatcher`
  房间域协议分发器。
- `IsRecording`
  当前是否正在录制。
- `CurrentTick`
  房间 Tick 计数器。
- `CreateTime`
  房间创建时间。
- `EmptySince`
  房间变为空房的时间。
- `ComponentIds`
  当前挂载的组件清单。
- `State`
  当前房间状态。

### 8.2 `Runtime/Transports/TCP/TcpTransportProvider.cs`

文件职责：

- 基于 `TcpClient/TcpListener` 的 TCP 传输实现。
- 生产环境重点链路之一。

关键设计点：

- 每个 TCP 连接都有独立的 4 字节长度前缀帧。
- `ReadExactAsync` 负责把“流式分片读取”拼成完整帧，因此不会把半包误交给上层。
- `TcpSendQueue` + `PendingSendFrame` 让每条连接同一时刻只有一个发送循环，避免高负载下每个包都起一个等待写流任务。
- 发送前先把 `Packet` 序列化成独立帧，避免上层提前归还池化 buffer 后后台任务读到脏数据。

内部类型：

- `TcpConnection`
  服务器端单连接包装：`Id`、`Client`、`SendQueue`。
- `TcpSendQueue`
  单连接发送上下文：`Stream`、`PendingFrames`、`SendLoopRunning`。
- `PendingSendFrame`
  已序列化完毕的一帧：`Buffer`、`Length`。

关键状态：

- `_serverListener`
  TCP 服务端监听器。
- `_serverConnections`
  当前所有服务端连接。
- `_client` / `_clientStream`
  当前 TCP 客户端连接与流。
- `_clientSendQueue`
  客户端发送队列。
- `_isPhysicalConnected`
  真实物理连接状态。

关键函数：

- `StartServer/StopServer`
  启停服务端监听。
- `AcceptClientsAsync`
  接受新连接并起收包循环。
- `ReceiveDataAsync`
  服务端单连接收包循环，按帧读满后解包。
- `StartClient/StopClient`
  启停客户端连接。
- `HandlePhysicalDisconnect`
  统一处理物理断线清理。
- `ReceiveClientDataAsync`
  客户端收包循环。
- `QueueSerializedSend`
  先序列化完整帧，再入单连接发送队列。
- `SendQueuedFramesAsync`
  串行写流。
- `DrainPendingFrames`
  回收待发送帧 buffer。
- `ReadExactAsync`
  从 TCP 流中读取固定长度字节。

### 8.3 `Runtime/Transports/KCP/KcpTransportProvider.cs`

文件职责：

- 基于 `kcp2k` 的可靠 UDP 传输实现。

关键特点：

- 同时实现 `INetworkTransport` 与 `IServerTransportPump`。
- 服务端由 `ServerRuntimeHost` 主动调用 `PumpServer()` 驱动 KCP。
- 客户端通过 `UnityPlayerLoopDispatcher` 注册周期泵。
- KCP 天生不是字节流协议，不存在 TCP 那种“读半帧”的流式粘拆包问题，但仍要依赖 `LitePacketFormatter` 保证上层 `Packet` 结构一致。

关键状态：

- `KcpConfig`
  当前 KCP 参数预设。
- `_server` / `_client`
  kcp2k 服务器和客户端。
- `_isServerActive` / `_isClientActive`
  逻辑活跃状态。
- `_isPhysicalConnected`
  真实物理连接状态。
- `_clientPumpRegistrationId`
  客户端周期泵注册 ID。
- `*_TotalPackets` / `*_DeserializeFailures`
  收包与解包统计。

关键函数：

- `PumpServer`
  服务端网络泵，由 `ServerRuntimeHost` 驱动。
- `StartServer/StopServer`
  启停 KCP 服务端。
- `OnServerDataReceived`
  服务端收包并直接向 `ServerApp` 上抛。
- `StartClient/StopClient`
  启停 KCP 客户端。
- `EnsureClientPumpRegistered/UnregisterClientPump`
  管理客户端周期泵。
- `OnClientDataReceived`
  客户端收包回主线程分发。

### 8.4 `Runtime/Transports/UDP/UdpTransportProvider.cs`

文件职责：

- 基于 `UdpClient` 的实验性 UDP 传输实现。

定位说明：

- 文件注释已经明确：只用于教学和实验，不建议承载生产业务主链。
- 没有像 KCP 那样的可靠性、重传、拥塞控制，也没有业务层会话恢复能力。

关键状态与函数：

- `_endpointToConnId` / `_connIdToEndpoint`
  用于把 `IPEndPoint` 映射成伪连接 ID。
- `ReceiveServerDataAsync`
  服务端 UDP 收包循环。
- `ReceiveClientDataAsync`
  客户端 UDP 收包循环。
- `HandlePhysicalDisconnect`
  统一处理客户端断开清理。
