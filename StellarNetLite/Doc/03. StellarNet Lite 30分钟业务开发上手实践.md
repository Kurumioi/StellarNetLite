# 03. StellarNet Lite 30分钟业务开发上手实践

本文档目标不是讲清楚所有细节，而是带你完成一条最短业务路径：

1. 新增协议。
2. 新增全局模块或房间组件。
3. 重新生成自动装配代码。
4. 在客户端或服务端看到功能生效。

---

## 1. 先决定功能属于哪一类

新增业务前先判断它属于 `Global` 还是 `Room`。

### 1.1 适合 `Global`

满足下面任一条件，通常写成全局模块：

- 不依赖房间上下文。
- 在大厅使用。
- 登录前后都可能调用。
- 需要跨房间统一管理。

示例：

- 账号登录
- 大厅聊天
- 拉房间列表
- 下载回放

### 1.2 适合 `Room`

满足下面任一条件，通常写成房间组件：

- 消息必须带房间上下文。
- 逻辑只对同房间成员生效。
- 需要随房间生命周期创建和销毁。

示例：

- 准备状态
- 开始游戏
- 房间内移动、战斗、聊天
- 房间内对象同步

---

## 2. 最短开发路线

建议按下面顺序做：

1. 先写协议。
2. 再写服务端处理器。
3. 再写客户端处理器。
4. 最后重新生成静态绑定代码。

不要一开始就写 UI。

---

## 3. 新增一个全局功能

下面以“大厅公告确认”为例说明全局功能写法。

### 3.1 定义协议

协议建议放在：

- `Shared/Protocol`

示例：

```csharp
[NetMsg(1501, NetScope.Global, NetDir.C2S)]
public sealed class C2S_AckAnnouncement : ILiteNetSerializable
{
    public string AnnouncementId;
}
```

如果服务端也要回包，再定义：

```csharp
[NetMsg(1502, NetScope.Global, NetDir.S2C)]
public sealed class S2C_AckAnnouncementResult : ILiteNetSerializable
{
    public bool Success;
}
```

### 3.2 写服务端模块

服务端全局模块通常放在：

- `Server/Modules`

示例：

```csharp
[ServerModule("ServerAnnouncementModule", "大厅公告模块")]
public sealed class ServerAnnouncementModule
{
    [NetHandler]
    public void OnC2S_AckAnnouncement(Session session, C2S_AckAnnouncement msg)
    {
        // 处理逻辑
    }
}
```

### 3.3 写客户端模块

客户端全局模块通常放在：

- `Client/Modules`

示例：

```csharp
[ClientModule("ClientAnnouncementModule", "大厅公告模块")]
public sealed class ClientAnnouncementModule
{
    [NetHandler]
    public void OnS2C_AckAnnouncementResult(S2C_AckAnnouncementResult msg)
    {
        // 处理逻辑
    }
}
```

### 3.4 重新生成静态绑定代码

执行：

- `StellarNetLite/强制重新生成协议与组件常量表`

不做这一步，新增模块和协议不会进运行时。

## 4. 新增一个房间功能

下面以“房间内表情消息”为例说明房间组件写法。

### 4.1 定义房间协议

```csharp
[NetMsg(1601, NetScope.Room, NetDir.C2S)]
public sealed class C2S_SendEmote : ILiteNetSerializable
{
    public int EmoteId;
}

[NetMsg(1602, NetScope.Room, NetDir.S2C)]
public sealed class S2C_EmoteSync : ILiteNetSerializable
{
    public string SessionId;
    public int EmoteId;
}
```

### 4.2 写服务端房间组件

```csharp
[RoomComponent(300, "RoomEmote", "房间表情组件")]
public sealed class ServerRoomEmoteComponent : ServerRoomComponent
{
    [NetHandler]
    public void OnC2S_SendEmote(Session session, C2S_SendEmote msg)
    {
        Room.BroadcastMessage(new S2C_EmoteSync
        {
            SessionId = session.SessionId,
            EmoteId = msg.EmoteId
        });
    }
}
```

### 4.3 写客户端房间组件

```csharp
[RoomComponent(300, "RoomEmote", "房间表情组件")]
public sealed class ClientRoomEmoteComponent : ClientRoomComponent
{
    [NetHandler]
    public void OnS2C_EmoteSync(S2C_EmoteSync msg)
    {
        Room.NetEventSystem.Broadcast(msg);
    }
}
```

### 4.4 把组件挂进房间模板

如果希望创建该房间时自动带上新组件，就要把组件 Id 加进房间模板或建房配置。

当前示例模板入口在：

- `Samples/SocialDemo/Shared/Binders/RoomTypeTemplateRegistry.cs`

### 4.5 重新生成自动装配代码

仍然要执行：

- `StellarNetLite/强制重新生成协议与组件常量表`

## 5. 用脚手架快速生成样板

如果不想手写目录和基础文件，可以直接使用：

- `StellarNetLite/业务脚手架生成器`

它可以生成两类样板：

- 房间组件
- 全局模块

建议使用方式：

1. 先确定功能名、协议 Id、组件 Id。
2. 用脚手架生成基础文件。
3. 再手动补业务逻辑。
4. 最后重新生成自动装配代码。

---

## 6. 开发后怎样验证

新增功能后，至少做下面几步验证：

1. Unity Console 中没有自动装配相关报错。
2. `MsgIdConst` 和 `ComponentIdConst` 中能看到新增项。
3. 客户端发送协议时没有 `未找到静态网络元数据` 报错。
4. 服务端能命中对应 `[NetHandler]`。
5. 回房间或回大厅后对象生命周期正常。

---

## 7. 最容易踩的坑

### 7.1 改了协议但没重新生成

现象通常是：

- 发包没响应。
- 收包不分发。
- 模块或组件根本没被注册。

### 7.2 把房间逻辑写进全局模块

后果通常是：

- 自己维护房间上下文。
- 生命周期容易混乱。
- 重连和回放难以接入。

### 7.3 在最终进房确认前发送房间消息

当前框架默认会拦截这类消息。

所以正确顺序一定是：

1. 创建 `ClientRoom`
2. 装配组件
3. 发 `C2S_RoomSetupReady`
4. 等 `S2C_RoomSetupResult.Success == true`
5. 再发送房间域业务消息
