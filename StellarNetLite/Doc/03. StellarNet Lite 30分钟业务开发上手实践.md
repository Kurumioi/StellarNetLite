# 03. StellarNet Lite 30分钟业务开发上手实践

> 本文不依赖任何 Demo。
> 
> 目标只有一个：让你从 0 开始，按框架基础能力，手动做出一套可运行的：
> 
> - 启动器
> - 全局模块
> - 房间业务组件
> - 客户端发送与监听
> - 服务端监听与回包

本文所有示例都只依赖框架本体：

- `Runtime`
- `Extensions`
- `StellarNetLiteGenerated`

不会依赖：

- `Samples/SocialDemo`
- Demo 的 `GameLauncher`
- Demo 的 `Panel`
- Demo 的 `Router`
- Demo 的任何 UI 逻辑

---

## 1. 做完本文后你应该能得到什么

做完本文后，你应该能独立完成下面这条完整链路：

1. 自己写一个启动器，启动 `Client` / `Server` / `Host`
2. 自己定义一组 `Global` 协议
3. 自己实现一组服务端全局模块和客户端全局模块
4. 自己定义一组 `Room` 协议
5. 自己实现一组服务端房间业务组件和客户端房间业务组件
6. 在客户端直接发送协议
7. 在客户端直接监听协议对象本身
8. 通过建房时的 `RoomDTO.ComponentIds` 把房间业务组件挂进房间
9. 用框架现成的默认房间流，跑通“登录 -> 建房 -> 进房 -> 房间业务消息”

本文故意不用复杂业务，只做两个最小案例：

- 全局模块案例：`Announcement`
- 房间组件案例：`RoomNotice`

它们足够小，但已经覆盖了最重要的开发主线：

`Launcher -> Shared 协议 -> Server 权威处理 -> Client 接收 -> 直接广播协议对象 -> 外部业务监听`

## 2. 先记住框架里最重要的 4 个角色

### 2.1 `StellarNetAppManager`

这是统一运行时入口。

它负责：

- 挂载并驱动传输层
- 读取并应用 `NetConfig`
- 创建 `ServerApp`
- 创建 `ClientApp`
- 装配自动注册出来的模块和房间组件

你自己写的启动器，本质上就是围绕它工作。

### 2.2 `ServerApp`

这是服务端主状态机。

它负责：

- 全局协议分发
- 会话管理
- 房间创建/销毁
- 给会话发全局消息

全局模块拿的是 `ServerApp`。

### 2.3 `Room`

这是服务端房间实例。

它负责：

- 房间成员管理
- 房间域协议分发
- 房间组件驱动
- 房间内广播

房间业务组件拿的是 `Room`。

### 2.4 `ClientApp` / `ClientRoom`

客户端主状态机是 `ClientApp`，当前房间实例是 `ClientRoom`。

客户端代码里要区分两类监听：

- 全局协议：`GlobalTypeNetEvent`
- 房间协议：`NetClient.CurrentRoom.NetEventSystem`

推荐做法很简单：

- 全局模块收到 `S2C_*` 后直接 `GlobalTypeNetEvent.Broadcast(msg)`
- 房间组件收到 `S2C_*` 后直接 `Room.NetEventSystem.Broadcast(msg)`

也就是说，外部业务层直接监听协议对象本身，不再额外包一层业务本地事件。

## 4. 第一步：从 0 写一个启动器

这个启动器只做框架基础工作，不做任何 Demo UI 初始化。

文件：

- `Assets/Scripts/MyGame/Runtime/BasicNetLauncher.cs`

代码：

```csharp
using System;
using StellarNet.Lite.Runtime;
using StellarNet.Lite.Shared.Infrastructure;
using UnityEngine;

namespace MyGame.Net
{
    public enum BasicNetMode
    {
        None,
        Client,
        Server,
        Host
    }

    public sealed class BasicNetLauncher : MonoBehaviour
    {
        [SerializeField] private StellarNetAppManager appManager;
        [SerializeField] private BasicNetMode netMode = BasicNetMode.Client;

        private void OnEnable()
        {
            StellarNetAppManager.OnClientStartedEvent += OnClientStarted;
            StellarNetAppManager.OnClientConnectedEvent += OnClientConnected;
            StellarNetAppManager.OnClientDisconnectedEvent += OnClientDisconnected;

            StellarNetAppManager.OnServerStartedEvent += OnServerStarted;
            StellarNetAppManager.OnServerStoppedEvent += OnServerStopped;
        }

        private void OnDisable()
        {
            StellarNetAppManager.OnClientStartedEvent -= OnClientStarted;
            StellarNetAppManager.OnClientConnectedEvent -= OnClientConnected;
            StellarNetAppManager.OnClientDisconnectedEvent -= OnClientDisconnected;

            StellarNetAppManager.OnServerStartedEvent -= OnServerStarted;
            StellarNetAppManager.OnServerStoppedEvent -= OnServerStopped;
        }

        private async void Start()
        {
            if (appManager == null)
            {
                Debug.LogError("BasicNetLauncher: appManager is null.");
                return;
            }

            NetConfig config = await NetConfigLoader.LoadRuntimeConfigAsync();
            appManager.ApplyConfig(config);

            switch (netMode)
            {
                case BasicNetMode.Client:
                    appManager.StartClient();
                    break;
                case BasicNetMode.Server:
                    appManager.StartServer();
                    break;
                case BasicNetMode.Host:
                    appManager.StartHost();
                    break;
            }
        }

        private void OnClientStarted()
        {
            Debug.Log("Client logic started.");
        }

        private void OnClientConnected()
        {
            Debug.Log("Client connected.");
        }

        private void OnClientDisconnected()
        {
            Debug.Log("Client disconnected.");
        }

        private void OnServerStarted()
        {
            Debug.Log("Server started.");
        }

        private void OnServerStopped()
        {
            Debug.Log("Server stopped.");
        }
    }
}
```

### 4.1 这个启动器依赖什么

你场景里至少要有一个 GameObject：

1. 挂 `StellarNetAppManager`
2. 再挂一个实现了 `INetworkTransport` 的传输层组件

例如：

- `KcpTransportProvider`
- `TcpTransportProvider`
- `UdpTransportProvider`

也就是说，启动器本身不直接处理网络包，它只负责：

- 加载配置
- 启动模式
- 订阅基础生命周期事件

## 5. 第二步：先定义一组全局协议

我们先做一个最小全局功能：公告请求。

文件：

- `Assets/Scripts/MyGame/Shared/Protocol/AnnouncementProtocols.cs`

代码：

```csharp
using StellarNet.Lite.Shared.Core;

namespace MyGame.Net
{
    [NetMsg(8201, NetScope.Global, NetDir.C2S)]
    public sealed class C2S_RequestAnnouncement
    {
        public string Content;
    }

    [NetMsg(8202, NetScope.Global, NetDir.S2C)]
    public sealed class S2C_AnnouncementSync
    {
        public string SenderName;
        public string Content;
    }
}
```

### 5.1 这一步的规则

一定要记住：

1. 协议定义放 `Shared`
2. 全局功能用 `NetScope.Global`
3. 客户端发服务端用 `NetDir.C2S`
4. 服务端回客户端用 `NetDir.S2C`

不要把协议类写进 `Client` 或 `Server`。

## 6. 第三步：写服务端全局模块

文件：

- `Assets/Scripts/MyGame/Server/Modules/ServerAnnouncementModule.cs`

代码：

```csharp
using StellarNet.Lite.Server.Core;
using StellarNet.Lite.Shared.Core;

namespace MyGame.Net
{
    [ServerModule("ServerAnnouncementModule", "公告模块")]
    public sealed class ServerAnnouncementModule
    {
        private readonly ServerApp _app;

        public ServerAnnouncementModule(ServerApp app)
        {
            _app = app;
        }

        [NetHandler]
        public void OnC2S_RequestAnnouncement(Session session, C2S_RequestAnnouncement msg)
        {
            if (session == null || msg == null)
            {
                return;
            }

            if (string.IsNullOrWhiteSpace(msg.Content))
            {
                return;
            }

            string senderName = string.IsNullOrWhiteSpace(session.AccountId)
                ? session.SessionId
                : session.AccountId;

            var syncMsg = new S2C_AnnouncementSync
            {
                SenderName = senderName,
                Content = msg.Content.Trim()
            };

            foreach (var kv in _app.Sessions)
            {
                Session target = kv.Value;
                if (target != null && target.IsOnline)
                {
                    _app.SendMessageToSession(target, syncMsg);
                }
            }
        }
    }
}
```

### 6.1 你需要理解的重点

全局模块的上下文是 `ServerApp`，不是 `Room`。

这意味着它适合做：

- 登录后大厅业务
- 公告
- 全局聊天
- 邮件入口
- 在线玩家列表

如果你的功能必须依附具体房间，就不要写在全局模块里。

## 7. 第四步：写客户端全局模块

文件：

- `Assets/Scripts/MyGame/Client/Modules/ClientAnnouncementModule.cs`

代码：

```csharp
using StellarNet.Lite.Client.Core.Events;
using StellarNet.Lite.Shared.Core;

namespace MyGame.Net
{
    [ClientModule("ClientAnnouncementModule", "公告模块")]
    public sealed class ClientAnnouncementModule
    {
        [NetHandler]
        public void OnS2C_AnnouncementSync(S2C_AnnouncementSync msg)
        {
            if (msg == null)
            {
                return;
            }

            GlobalTypeNetEvent.Broadcast(msg);
        }
    }
}
```

### 7.1 为什么这里直接广播协议对象

推荐做法是：

- 不额外定义 `Local_AnnouncementReceived`
- 直接广播 `S2C_AnnouncementSync`

这样外部业务层只需要记住一套名字：

- 协议名

而不是两套：

- 协议名
- 本地转发事件名

## 9. 第六步：写服务端房间业务组件

文件：

- `Assets/Scripts/MyGame/Server/Components/ServerRoomNoticeComponent.cs`

代码：

```csharp
using StellarNet.Lite.Server.Core;
using StellarNet.Lite.Shared.Core;

namespace MyGame.Net
{
    [RoomComponent(930, "RoomNotice", "房间通知组件")]
    public sealed class ServerRoomNoticeComponent : ServerRoomComponent
    {
        [NetHandler]
        public void OnC2S_SendRoomNotice(Session session, C2S_SendRoomNotice msg)
        {
            if (Room == null || session == null || msg == null)
            {
                return;
            }

            if (string.IsNullOrWhiteSpace(msg.Content))
            {
                return;
            }

            Room.BroadcastMessage(new S2C_RoomNoticeSync
            {
                SenderSessionId = session.SessionId,
                Content = msg.Content.Trim()
            });
        }
    }
}
```

### 9.1 为什么这里拿的是 `Room`

因为房间业务组件本来就属于某个房间实例。

它适合处理：

- 房间聊天
- 房间通知
- 房间准备
- 房间战斗
- 房间交互

它天然有：

- `Room`
- 当前房间成员
- 当前房间状态

## 11. 第八步：写一个最小客户端控制脚本

前面只是定义了“框架如何接消息”，现在还要有地方主动发消息。

文件：

- `Assets/Scripts/MyGame/Client/Runtime/BasicClientConsole.cs`

代码：

```csharp
using StellarNet.Lite.Client.Core;
using StellarNet.Lite.Client.Core.Events;
using StellarNet.Lite.Shared.Protocol;
using UnityEngine;

namespace MyGame.Net
{
    public sealed class BasicClientConsole : MonoBehaviour
    {
        [SerializeField] private string accountId = "player_001";
        [SerializeField] private string clientVersion = "0.0.1";

        private void OnEnable()
        {
            GlobalTypeNetEvent.Register<S2C_LoginResult>(OnLoginResult).UnRegisterWhenMonoDisable(this);
            GlobalTypeNetEvent.Register<S2C_AnnouncementSync>(OnAnnouncementSync).UnRegisterWhenMonoDisable(this);
            GlobalTypeNetEvent.Register<S2C_CreateRoomResult>(OnCreateRoomResult).UnRegisterWhenMonoDisable(this);
            GlobalTypeNetEvent.Register<S2C_JoinRoomResult>(OnJoinRoomResult).UnRegisterWhenMonoDisable(this);
            GlobalTypeNetEvent.Register<S2C_RoomSetupResult>(OnRoomSetupResult).UnRegisterWhenMonoDisable(this);
        }

        private void Update()
        {
            if (Input.GetKeyDown(KeyCode.L))
            {
                Login();
            }

            if (Input.GetKeyDown(KeyCode.Alpha1))
            {
                RequestAnnouncement();
            }

            if (Input.GetKeyDown(KeyCode.Alpha2))
            {
                CreateRoomWithRoomNotice();
            }

            if (Input.GetKeyDown(KeyCode.Alpha3))
            {
                SendRoomNotice();
            }
        }

        public void Login()
        {
            NetClient.Send(new C2S_Login
            {
                AccountId = accountId,
                ClientVersion = clientVersion
            });
        }

        public void RequestAnnouncement()
        {
            NetClient.Send(new C2S_RequestAnnouncement
            {
                Content = "hello from global module"
            });
        }

        public void CreateRoomWithRoomNotice()
        {
            NetClient.Send(new C2S_CreateRoom
            {
                RoomConfig = new RoomDTO
                {
                RoomName = "MyFirstRoom",
                MaxMembers = 8,
                EnableReplayRecording = false,
                Password = string.Empty,
                CustomProperties = new System.Collections.Generic.Dictionary<string, string>(),
                ComponentIds = new[]
                    {
                        ComponentIdConst.RoomSettings,
                        ComponentIdConst.RoomNotice
                    }
                }
            });
        }

        public void SendRoomNotice()
        {
            if (NetClient.CurrentRoom == null)
            {
                Debug.LogWarning("CurrentRoom is null.");
                return;
            }

            NetClient.Send(new C2S_SendRoomNotice
            {
                Content = "hello from room component"
            });
        }

        private void OnLoginResult(S2C_LoginResult msg)
        {
            Debug.Log($"LoginResult success={msg.Success} reason={msg.Reason}");
        }

        private void OnAnnouncementSync(S2C_AnnouncementSync msg)
        {
            Debug.Log($"Announcement sender={msg.SenderName} content={msg.Content}");
        }

        private void OnCreateRoomResult(S2C_CreateRoomResult msg)
        {
            Debug.Log($"CreateRoomResult success={msg.Success} roomId={msg.RoomId} reason={msg.Reason}");
        }

        private void OnJoinRoomResult(S2C_JoinRoomResult msg)
        {
            Debug.Log($"JoinRoomResult success={msg.Success} roomId={msg.RoomId} reason={msg.Reason}");
        }

        private void OnRoomSetupResult(S2C_RoomSetupResult msg)
        {
            Debug.Log($"RoomSetupResult success={msg.Success} roomId={msg.RoomId} reason={msg.Reason}");

            if (!msg.Success || NetClient.CurrentRoom == null)
            {
                return;
            }

            NetClient.CurrentRoom.NetEventSystem
                .Register<S2C_RoomNoticeSync>(OnRoomNoticeSync)
                .UnRegisterWhenMonoDisable(this);
        }

        private void OnRoomNoticeSync(S2C_RoomNoticeSync msg)
        {
            Debug.Log($"RoomNotice sender={msg.SenderSessionId} content={msg.Content}");
        }
    }
}
```

### 11.1 这个脚本做了什么

它没有任何 Demo UI 依赖，只靠：

- `Input.GetKeyDown`
- `Debug.Log`

就可以验证整条链路。

按键说明：

- `L`：登录
- `1`：发送全局公告请求
- `2`：创建一个只挂 `RoomSettings + RoomNotice` 的房间
- `3`：在当前房间发一条房间通知

## 12. 第九步：为什么房间组件不是写完就能生效

这是最容易踩坑的一步。

你写了：

- `[RoomComponent(930, "RoomNotice", ...)]`

并不代表所有房间都会自动带 `RoomNotice`。

房间组件要真正生效，必须出现在某个房间的 `ComponentIds` 里。

最小做法就是像本文这样，在建房时直接写：

```csharp
ComponentIds = new[]
{
    ComponentIdConst.RoomSettings,
    ComponentIdConst.RoomNotice
}
```

也就是说：

- 全局模块靠注册自动生效
- 房间业务组件靠“注册 + 挂进房间”才生效

## 13. 第十步：新增代码后必须重新生成

当你新增了下面任意内容后，都必须重新生成：

- `[NetMsg]`
- `[ServerModule]`
- `[ClientModule]`
- `[RoomComponent]`
- `[NetHandler]`

你至少要重新生成出这些内容：

- `MsgIdConst`
- `ComponentIdConst`
- 自动注册表
- 自动绑定器

菜单入口：

- `StellarNetLite/重新生成协议与组件常量表`
- `StellarNetLite/强制重新生成协议与组件常量表`

如果你忘了这一步，最常见的表现就是：

- 消息发得出去但收不到
- 新组件写了但房间里没有
- 常量表里没有新协议 Id / 组件 Id

## 16. 你最容易写错的地方

### 16.1 把协议放错目录

协议一定放 `Shared`。

### 16.2 把全局功能写成房间组件

结果就是功能离开房间就失效。

### 16.3 把房间功能写成全局模块

结果就是拿不到 `Room` 上下文。

### 16.4 忘了把房间组件挂进 `ComponentIds`

这会导致组件根本不被创建。

### 16.5 忘了重新生成

这是最常见的“代码都写了但没反应”来源。

### 16.6 在 `S2C_RoomSetupResult.Success == true` 之前急着发房间消息

当前框架会在最终确认前拦截 `NetScope.Room` 发送。

### 16.7 客户端额外包一层业务本地事件

不推荐。

推荐直接广播协议对象本身，让业务层直接监听协议对象。

---

## 17. 一句话总结本文

在不依赖任何 Demo 的前提下，StellarNet Lite 最稳妥的业务开发主线就是：

`Launcher 启动框架 -> Shared 定协议 -> Server 写权威逻辑 -> Client 直接广播协议对象 -> 房间组件通过 ComponentIds 挂进房间 -> 重新生成 -> 运行验证`
