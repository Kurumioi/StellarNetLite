# 14. StellarNet Lite 源码总索引与阅读地图

本文档不是“怎么用”的教程，而是“源码从哪里看、每一层负责什么、查某个类型该去哪里”的总索引。

本轮源码级文档覆盖范围如下：

- `StellarNetLite/Runtime`
- `StellarNetLite/Extensions`
- `StellarNetLite/Editor`
- `StreamingAssets/StellarNetLiteLoadTest`

本轮不把以下内容当作“框架源码 API 参考”的主体：

- `Samples/SocialDemo`
- 纯资源文件，例如预制体、图片、材质、模型
- `StellarNetLiteGenerated` 的每一行生成代码本体

原因很简单：用户后续维护、扩展、排查问题时，最常读的是运行时骨架、官方扩展、编辑器工具链和压测工具，而不是资源资产本身。

## 2. 框架源码分层图

可以把框架理解成下面这几层：

### 2.1 `Runtime`

这是底层运行时骨架，解决的是：

- 统一启动入口是谁
- 客户端/服务端主状态机如何存在
- 包结构、协议元信息、序列化、日志、配置怎么统一
- 房间和会话如何建模
- 收包后如何路由到全局模块或房间组件
- 传输层如何把物理连接抽象成统一接口

`Runtime` 不直接写死登录、建房、大厅、对象同步这些业务流程。

### 2.2 `Extensions`

这是官方默认扩展层，解决的是：

- 默认登录和重连链
- 默认大厅和房间流程
- 默认对象同步方案
- 默认回放录制/下载/回放方案
- 默认弱网监控方案

也就是说，`Runtime` 负责“框架骨架”，`Extensions` 负责“官方默认玩法实现”。

### 2.3 `Editor`

这是把“源码声明”变成“运行时可装配表”的工具链，主要负责：

- 扫描 `[NetMsg]` / `[NetHandler]`
- 扫描 `[RoomComponent]` / `[ServerModule]` / `[ClientModule]`
- 扫描网络预制体
- 生成运行时依赖的静态注册表
- 提供配置编辑、脚手架生成、服务端监控、压测面板

### 2.4 `StreamingAssets/StellarNetLiteLoadTest`

这是一个独立 .NET 压测工具，不依赖 Unity 场景运行，主要负责：

- 批量起机器人
- 自动跑登录/建房/进房/Ready/StartGame
- 周期性发移动、动作、气泡行为
- 支持 `kcp` 和 `tcp`
- 支持运行时动态加房、删房、结束房间

## 3. 核心调用链地图

### 3.1 启动链

1. Unity 场景里挂载 `StellarNetAppManager`
2. `Awake` 中装配 `INetworkTransport`、`LiteNetSerializer`、`NetConfig`
3. `NetMessageMapper.Initialize()` 载入自动生成的协议元表
4. `AutoRegistry` 把模块、组件、处理器注册到 `ClientApp` / `ServerApp`
5. `GeneratedRuntimeAccess` 创建运行时扩展桥
6. 传输层启动后，`StellarNetAppManager` 再分别接客户端宿主和服务端宿主

### 3.2 服务端收包链

1. 物理层收到包
2. `INetworkTransport.OnServerReceivePacketEvent`
3. `StellarNetAppManager.HandleServerReceivePacket`
4. `ServerRuntimeHost.ExecuteOrEnqueue`
5. `ServerApp.OnReceivePacket`
6. 按 `NetScope` 进入：
   - `GlobalDispatcher`
   - `RoomScheduler` / `Room.DispatchPacket`
7. 自动生成绑定器把包分发到：
   - `ServerModule` 的 `[NetHandler]`
   - `ServerRoomComponent` 的 `[NetHandler]`

### 3.3 客户端收包链

1. 物理层收到包
2. `INetworkTransport.OnClientReceivePacketEvent`
3. `StellarNetAppManager.HandleClientReceivePacket`
4. 运行时扩展桥先拿到包做附加处理
5. `ClientApp.OnReceivePacket`
6. 按 `NetScope` 进入：
   - `ClientGlobalDispatcher`
   - `CurrentRoom.Dispatcher`
7. 自动生成绑定器把包分发到：
   - `ClientModule` 的 `[NetHandler]`
   - `ClientRoomComponent` 的 `[NetHandler]`

### 3.4 进房链

1. 客户端发 `C2S_CreateRoom` 或 `C2S_JoinRoom`
2. 服务端模块只做“授权”或“建房”，先返回 `CreateRoomResult` / `JoinRoomResult`
3. 客户端据此本地创建 `ClientRoom`，并按 `ComponentIds` 装配组件
4. 客户端发 `C2S_RoomSetupReady`
5. 服务端最终调用 `Room.TryAddMember` 或 `Room.ResumeMember`
6. 服务端回 `S2C_RoomSetupResult`
7. 只有拿到成功的 `RoomSetupResult`，客户端才算真正进入房间

这个两段式流程是整个框架很重要的边界，很多“为什么房间消息发不出去”的问题都与这里有关。

## 4. 目录级索引

### 4.1 `Runtime`

- `Client/Core`
  客户端状态机、房间、会话、分发器、事件系统、网络门面。
- `Server/Core`
  服务端状态机、会话、房间、快照、分发器、组件工厂。
- `Server/Infrastructure`
  独立线程宿主、房间调度器、出站分发、持久化运行时、无头日志。
- `Shared/Core`
  协议特性、包结构、传输接口、协议索引器。
- `Shared/Infrastructure`
  配置、日志、主线程桥接、序列化抽象。
- `Transports`
  `TCP`、`KCP`、`UDP` 与公共包格式器。

### 4.2 `Extensions`

- `DefaultGameFlow`
  登录、大厅、建房、进房、离房、重连主链。
- `RoomFlow`
  房间基础成员/准备/房主/开局/结算流程。
- `ObjectSync`
  网络实体、变换同步、动画同步、回放快照消费与导出。
- `Replay`
  录像录制、存储、分块下载、客户端回放。
- `NetworkMonitoring`
  Ping、RTT、弱网告警、阻断、主动熔断。

### 4.3 `Editor`

- `LiteProtocolScanner`
  最核心的静态绑定生成器。
- `NetPrefabScanner`
  网络预制体扫描与常量生成器。
- `NetConfigEditorWindow`
  三套配置的统一编辑入口。
- `StellarNetScaffoldWindow`
  模块/组件脚手架。
- `ServerMonitorWindow`
  运行中房间/会话监控。
- `LoadTestWindow`
  压测工具启动器。
- `FolderContentCopyTool`
  模板目录拷贝工具。

### 4.4 `StellarNetLiteLoadTest`

- `Program.cs`
  CLI 入口、参数、机器人总调度器。
- `Transports.cs`
  KCP/TCP 压测客户端传输实现。
- `NetworkPrimitives.cs`
  压测工具自带的包格式、协议 ID、简化 DTO、移动/动作/气泡序列化器。

## 5. 阅读建议

### 5.1 如果你要修框架主链

优先顺序：

1. `StellarNetAppManager`
2. `ClientApp`
3. `ServerApp`
4. `Room`
5. `ServerRuntimeHost`
6. `INetworkTransport` + 具体传输实现

### 5.2 如果你要加一个新业务模块

优先顺序：

1. `NetCoreDefines.cs` 里的特性约定
2. `LiteProtocolScanner.cs`
3. `ServerModule` / `ClientModule` 示例
4. `ServerRoomComponent` / `ClientRoomComponent` 示例
5. `AutoRegistry` 的生成消费点

### 5.3 如果你要查同步或回放问题

优先顺序：

1. `ServerObjectSyncComponent`
2. `ClientObjectSyncComponent`
3. `ReplayRoomRecordingService`
4. `ServerReplayStorage`
5. `ClientReplayPlayer`

### 5.4 如果你要查高负载或连接问题

优先顺序：

1. `TcpTransportProvider`
2. `KcpTransportProvider`
3. `ServerOutboundDispatcher`
4. `ServerRuntimeHost`
5. `ServerRoomScheduler`
6. `StellarNetLiteLoadTest`
