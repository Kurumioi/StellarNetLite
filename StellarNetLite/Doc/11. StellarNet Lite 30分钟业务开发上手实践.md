    # 11. StellarNet Lite 30分钟业务开发上手实践

> 本文不是再讲一遍概念。  
> 本文只解决一件事：**让一个普通 Unity 业务开发者，30 分钟内照着做出一个可跑的全局模块和一个可跑的房间业务组件。**

---

## 1. 这篇文档做完以后，你应该得到什么

做完本文后，你应该能独立完成下面两类业务开发：

1. 新增一个全局模块
2. 新增一个房间业务组件

本文用两个最小案例串起来：

- 全局模块案例：`Announcement` 公告模块
- 房间组件案例：`RoomNotice` 房间通知组件

这两个案例故意不依赖复杂 UI，也不改 Runtime 主链，重点是把开发者最关心的业务链路走通：

`Shared 协议定义 -> Server 权威处理 -> Client 接收转事件 -> 业务层监听 -> 运行验证`

---

## 2. 先记住一句话：什么时候写模块，什么时候写组件

如果你的功能：

- 不依赖某个具体房间上下文，用**全局模块**
- 必须附着在当前房间实例上，用**房间组件**

快速判断：

- 登录、公告、大厅列表、邮件、背包入口：全局模块
- 准备、开始游戏、房间聊天、房间内交互、战斗逻辑：房间组件

如果这一条还不稳，先回看：

- `Doc/03. StellarNet Lite 全局模块与房间组件开发指南.md`

---

## 3. 开工前先确认 4 件事

1. 你的协议定义放在 `Shared`
2. 服务端权威逻辑放在 `Server`
3. 客户端接收和事件派发放在 `Client`
4. 你知道新增完后要重新生成自动装配代码

对应参考：

- `Doc/01. StellarNet Lite 核心概念与全流程指南.md`
- `Doc/03. StellarNet Lite 全局模块与房间组件开发指南.md`
- `Doc/08. StellarNet Lite 代码生成与编辑器工具链指南.md`

如果你新增了协议、模块、房间组件，但没有重新生成，后面大概率会出现：

- 收不到消息
- 新组件挂不上
- `MsgIdConst` / `ComponentIdConst` 还是旧的

## 4. 实战 A：30 分钟里的第一个全局模块

这里做一个最小的“公告模块”。

目标链路：

1. 客户端发送 `C2S_RequestAnnouncement`
2. 服务端全局模块处理
3. 服务端广播 `S2C_AnnouncementSync`
4. 客户端收到后抛给全局事件
5. UI 或业务层自己监听显示

### 4.1 第一步：在 Shared 定义协议

建议新建：

- `Shared/Protocol/AnnouncementProtocols.cs`

代码示例：

```csharp
using StellarNet.Lite.Shared.Core;

[NetMsg(8201, NetScope.Global, NetDir.C2S)]
public sealed class C2S_RequestAnnouncement
{
    public string Content;
}

[NetMsg(8202, NetScope.Global, NetDir.S2C)]
public sealed class S2C_AnnouncementSync
{
    public string SenderName;
    public string Content;
}
```

这里最重要的是三件事：

- `NetScope.Global`
- 方向写对
- 协议定义放在 `Shared`

如果你把协议写进 `Client` 或 `Server`，后面生成链和双端对齐都会开始出问题。

### 4.2 第二步：在 Server 写权威处理

建议新建：

- `Server/Modules/ServerAnnouncementModule.cs`

代码示例：

```csharp
using StellarNet.Lite.Server.Core;

[ServerModule("ServerAnnouncementModule", "公告模块")]
public sealed class ServerAnnouncementModule
{
    private readonly ServerApp _app;

    public ServerAnnouncementModule(ServerApp app)
    {
        _app = app;
    }

    [NetHandler]
    public void OnC2S_RequestAnnouncement(Session session, C2S_RequestAnnouncement msg)
    {
        if (session == null || msg == null || string.IsNullOrWhiteSpace(msg.Content))
        {
            return;
        }

        string senderName = string.IsNullOrEmpty(session.AccountId)
            ? session.SessionId
            : session.AccountId;

        var syncMsg = new S2C_AnnouncementSync
        {
            SenderName = senderName,
            Content = msg.Content.Trim()
        };

        foreach (var kvp in _app.Sessions)
        {
            Session target = kvp.Value;
            if (target != null && target.IsOnline)
            {
                _app.SendMessageToSession(target, syncMsg);
            }
        }
    }
}
```

这个模块体现的是 StellarNet Lite 当前推荐的业务写法：

- 客户端只提请求
- 服务端读真实状态
- 服务端组装权威结果
- 客户端只接收和表现

### 4.3 第三步：在 Client 接收并转成事件

建议新建：

- `Client/Modules/ClientAnnouncementModule.cs`

代码示例：

```csharp
using StellarNet.Lite.Client.Core;
using StellarNet.Lite.Client.Core.Events;

[ClientModule("ClientAnnouncementModule", "公告模块")]
public sealed class ClientAnnouncementModule
{
    [NetHandler]
    public void OnS2C_AnnouncementSync(S2C_AnnouncementSync msg)
    {
        if (msg == null)
        {
            return;
        }

        GlobalTypeNetEvent.Broadcast(msg);
    }
}
```

这里的重点不是直接操作 UI，而是先把网络结果转成事件。

这样后面你的 UI、Router、Presenter、Panel 都可以自己订阅，不会把业务逻辑写死在网络层里。

### 4.4 第四步：业务层怎么触发

任意按钮或业务入口里都可以这样发：

```csharp
NetClient.Send(new C2S_RequestAnnouncement
{
    Content = "Hello StellarNet Lite"
});
```

如果你要做 UI 展示，就在业务表现层监听：

```csharp
GlobalTypeNetEvent.Subscribe<S2C_AnnouncementSync>(OnAnnouncementSync);
```

## 5. 实战 B：30 分钟里的第一个房间业务组件

这里做一个最小的“房间通知组件”。

目标链路：

1. 客户端在房间内发送 `C2S_SendRoomNotice`
2. 服务端房间组件处理
3. 服务端广播 `S2C_RoomNoticeSync`
4. 客户端房间组件收到后抛给当前房间事件系统
5. 当前房间的 UI 或 View 监听显示

### 5.1 第一步：在 Shared 定义房间协议

建议新建：

- `Shared/Protocol/RoomNoticeProtocols.cs`

代码示例：

```csharp
using StellarNet.Lite.Shared.Core;

[NetMsg(9301, NetScope.Room, NetDir.C2S)]
public sealed class C2S_SendRoomNotice
{
    public string Content;
}

[NetMsg(9302, NetScope.Room, NetDir.S2C)]
public sealed class S2C_RoomNoticeSync
{
    public string SenderSessionId;
    public string Content;
}
```

这里最关键的是：

- `NetScope.Room`
- 它是房间域协议，不是全局域协议

### 5.2 第二步：在 Server 写房间权威组件

建议新建：

- `Server/Components/ServerRoomNoticeComponent.cs`

代码示例：

```csharp
using StellarNet.Lite.Server.Core;

[RoomComponent(930, "RoomNotice", "房间通知组件")]
public sealed class ServerRoomNoticeComponent : ServerRoomComponent
{
    [NetHandler]
    public void OnC2S_SendRoomNotice(Session session, C2S_SendRoomNotice msg)
    {
        if (session == null || msg == null || string.IsNullOrWhiteSpace(msg.Content))
        {
            return;
        }

        Room.BroadcastMessage(new S2C_RoomNoticeSync
        {
            SenderSessionId = session.SessionId,
            Content = msg.Content.Trim()
        });
    }
}
```

这里和全局模块的核心区别只有一个：

- 全局模块看的是 `ServerApp`
- 房间组件看的是 `Room`

### 5.3 第三步：在 Client 写房间组件

建议新建：

- `Client/Components/ClientRoomNoticeComponent.cs`

代码示例：

```csharp
using StellarNet.Lite.Client.Core;

[RoomComponent(930, "RoomNotice", "房间通知组件")]
public sealed class ClientRoomNoticeComponent : ClientRoomComponent
{
    [NetHandler]
    public void OnS2C_RoomNoticeSync(S2C_RoomNoticeSync msg)
    {
        if (msg == null || Room == null)
        {
            return;
        }

        Room.NetEventSystem.Broadcast(msg);
    }
}
```

房间内业务更推荐抛给：

- `Room.NetEventSystem`

而不是抛给全局事件系统。

因为它天然跟当前房间实例同生命周期，房间销毁后不会把事件继续挂在大厅态里。

## 6. 新增了房间组件，不代表它已经会自动进房

这是初学者最容易漏的一步。

你写了 `RoomNotice` 组件，只代表它可以被扫描、注册、创建；不代表每个房间模板都会自动挂它。

你还要把它加到房间模板里。

当前 Demo 可直接参考：

- `Samples/SocialDemo/Shared/Binders/RoomTypeTemplateRegistry.cs`

这个文件当前定义了 Demo 房间模板，例如：

- `SocialRoom`
- `RoomSettings`
- `ObjectSync`

如果你想让某个模板默认带 `RoomNotice`，就要把：

```csharp
ComponentIdConst.RoomNotice
```

加入对应模板的 `ComponentIds` 列表里。

示意：

```csharp
ComponentIds = new[]
{
    ComponentIdConst.SocialRoom,
    ComponentIdConst.RoomSettings,
    ComponentIdConst.ObjectSync,
    ComponentIdConst.RoomNotice
};
```

如果你没做这一步，代码虽然写完了，但房间实例里根本没有你的组件。

## 8. 新增完协议和组件后，一定要重新生成

这是当前框架里最重要的“别跳步”环节之一。

新增下面任何内容后，都要重新生成：

- `[NetMsg]`
- `[RoomComponent]`
- `[ServerModule]`
- `[ClientModule]`
- `[NetHandler]`

对应说明看：

- `Doc/08. StellarNet Lite 代码生成与编辑器工具链指南.md`

你至少要确认这些生成结果已经更新：

- `MsgIdConst`
- `ComponentIdConst`
- 自动注册表
- 自动绑定器

如果你本地还没生成过，那么仓库里看不到 `StellarNetLiteGenerated` 是正常的；需要在 Unity Editor 里重新跑生成链。

## 9. 当前版本和“房间确认闭环”有关的一个新注意点

现在房间主链已经补上了最终确认闭环。

普通进房链路已经变成：

1. 服务端授权进入房间
2. 客户端初始化房间对象和房间组件
3. 客户端发送 `C2S_RoomSetupReady`
4. 服务端返回 `S2C_RoomSetupResult`
5. 只有成功后，才算正式进入稳定房间态

这意味着一条很重要的业务规则：

- 在 `S2C_RoomSetupResult.Success == true` 之前，客户端会自动拦截 `NetScope.Room` 发送

对业务开发者来说，这反而是好事：

- 你不用自己补一层“房间未完成初始化，别发 room-scope 包”的底层防线

但你要知道这个行为存在，否则你会误以为“为什么刚建完房立刻发房间消息没有效果”。

正确做法是：

- 等房间真正进入稳定态后，再触发房间内业务发送

## 10. 最小验证路径

如果你按本文加完了一个全局模块和一个房间组件，最小验证建议这样走：

1. 启动服务端和客户端
2. 登录
3. 触发一次 `Announcement` 全局消息发送
4. 确认客户端能收到 `S2C_AnnouncementSync`
5. 创建或进入一个房间
6. 确认进房完成，房间处于稳定态
7. 触发一次 `RoomNotice` 房间消息发送
8. 确认房间内成员都能收到 `S2C_RoomNoticeSync`

如果你基于 Demo 验证，推荐顺手看：

- 房间模板里是否真的挂上了你的组件
- 当前房间 UI / View 是否正确监听了 `Room.NetEventSystem`

## 11. 初学者最常见的 8 个错误

### 11.1 协议写错目录

协议应该放在 `Shared`，不要直接写到 `Client` 或 `Server`。

### 11.2 该写全局模块的东西写成房间组件

结果通常是：

- 不在房间里功能就失效
- 生命周期不对

### 11.3 该写房间组件的东西写成全局模块

结果通常是：

- 业务拿不到当前房间上下文
- 广播目标和状态边界会混乱

### 11.4 忘了把房间组件挂进模板

这是最容易被忽略的一步。

### 11.5 新增完代码但没重新生成

这是最常见的“代码明明写了，但运行就是没反应”的来源。

### 11.6 客户端直接改权威状态

客户端更适合做：

- 输入
- 缓存
- 表现
- 事件转发

### 11.7 房间刚初始化就急着发 room-scope 包

当前版本在最终确认前会自动拦截。

### 11.8 把网络处理和 UI 强绑在一起

更推荐：

`网络层 -> 事件 -> 表现层`

这样后面你切在线 / 回放 / 重连 / 不同 UI 形态时成本才不会爆炸。

## 13. 一句话总结本文

在 StellarNet Lite 里，普通业务开发者新增一个功能，最稳定的主线就是：

`Shared 定协议 -> Server 写权威处理 -> Client 转事件 -> 房间组件挂模板 -> 重新生成 -> 运行验证`
