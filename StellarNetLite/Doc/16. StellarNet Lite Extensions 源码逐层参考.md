# 16. StellarNet Lite Extensions 源码逐层参考

本文档覆盖 `StellarNetLite/Extensions`。

理解 `Extensions` 的关键前提：

- `Runtime` 解决“框架骨架”。
- `Extensions` 解决“官方默认业务能力”。
- 这层代码大多通过三种方式接入 Runtime：
  - `ServerModule` / `ClientModule`
  - `ServerRoomComponent` / `ClientRoomComponent`
  - `RuntimeFeatureBridgeBase`

---

## 1. DefaultGameFlow

这一组脚本负责默认登录、大厅、建房、加房、离房、挂起房间、重连。

### 1.1 `DefaultGameFlow/Runtime/DefaultGameFlowRuntimeFeature.cs`

文件职责：

- 默认游戏流程运行时桥。
- 把“登录前豁免协议、踢下线通知、在线列表刷新、重连补登”等流程从 Runtime 主体下沉到扩展层。

成员：

- `OnServerAppCreated`
  把 `DefaultRoomMembershipNotifier.Instance` 挂到 `ServerApp.RoomMembershipNotifier`。
- `OnServerSessionStateChanged`
  会话状态变化时广播大厅在线玩家列表。
- `TryNotifyServerSessionKick`
  服务端踢人前先发 `S2C_KickOut`。
- `OnClientAppCreated`
  注册弱网豁免协议：`C2S_Login`、`C2S_ConfirmReconnect`、`C2S_ReconnectReady`。
- `OnClientConnected`
  如果当前处于重连流程且有 `AccountId`，自动补发登录请求。

### 1.2 `DefaultGameFlow/Server/Infrastructure/DefaultRoomMembershipNotifier.cs`

文件职责：

- 默认房间成员通知器。
- 当房间因为超时清理、强制结算等原因把玩家移出时，默认发送 `S2C_LeaveRoomResult`。

成员：

- `Instance`
  单例。
- `NotifyForcedLeave(Room room, Session session)`
  调用 `room.SendMessageTo` 通知目标成员。

### 1.3 `DefaultGameFlow/Server/Modules/ServerUserModule.cs`

文件职责：

- 服务端用户鉴权与重连模块。

核心字段：

- `_app`
  当前 `ServerApp`。

核心处理函数：

- `OnC2S_Login`
  处理登录：
  - 校验 `AccountId`
  - 校验 `ClientVersion`
  - 处理同账号重登/顶号
  - 旧会话重绑连接
  - 首次登录创建新鉴权会话
  - 决定是否存在可恢复房间
- `OnC2S_ConfirmReconnect`
  决定客户端是否接受恢复房间。
- `OnC2S_ReconnectReady`
  客户端重连装配完成后真正恢复房间并补发快照。
- `TryGetRecoverableRoom`
  判断会话当前能否恢复到某个房间，同时清理失效恢复上下文。

为什么这个模块重要：

- 账号唯一性、顶号、重绑物理连接、恢复房间都在这里。
- 生产环境大多数“重连怎么失效了”的问题都从这里开始查。

### 1.4 `DefaultGameFlow/Server/Modules/ServerRoomModule.cs`

文件职责：

- 服务端房间生命周期模块。

核心字段：

- `_app`
  当前 `ServerApp`。

关键处理函数：

- `OnC2S_CreateRoom`
  建房：
  - 校验当前不在房间内
  - 校验 `RoomDTO`
  - 去重组件 ID
  - 生成房间 ID
  - 创建房间实例
  - 写入 `RoomConfigModel`
  - 调用 `ServerRoomFactory.BuildComponents`
  - 授权当前会话后续确认进入该房间
- `OnC2S_JoinOrCreateRoom`
  如果房间存在则授权加入，不存在则按给定 `RoomId` 降级建房。
- `OnC2S_JoinRoom`
  加房授权：
  - 校验房间是否存在、是否结束、是否满员、密码是否正确
  - 支持“恢复同一个挂起房间”的特殊路径
- `OnC2S_RoomSetupReady`
  最终进房确认：
  - 校验 `AuthorizedRoomId`
  - 处理新进房与恢复进房两条链
  - 对恢复链走 `room.ResumeMember`
  - 对新进房走 `room.TryAddMember`
- `OnC2S_DisconnectRoom`
  挂起当前房间，保留恢复上下文。
- `OnC2S_LeaveRoom`
  离房，必要时销毁空房。
- `DeduplicateComponentIds`
  去重组件 ID 列表。
- `BroadcastRoomListToLobby`
  向所有大厅在线玩家刷新房间列表。
- `CleanupPreviousRecoverableRoom`
  用户进入新房间时清理旧恢复房间上下文。

### 1.5 `DefaultGameFlow/Server/Modules/ServerLobbyModule.cs`

文件职责：

- 服务端大厅信息与全局社交模块。

核心函数：

- `OnC2S_GetRoomList`
  返回当前大厅房间列表。
- `OnC2S_GlobalChat`
  处理大厅聊天并广播。
- `BuildRoomListResponse`
  把 `RoomRuntimeSnapshot` 转成 `RoomBriefInfo[]`。
- `BuildOnlinePlayerListResponse`
  把所有 `Session` 转成 `OnlinePlayerInfo[]`。
- `BroadcastOnlinePlayerList`
  向在线玩家全量推送在线列表。
- `BroadcastPlayerStateChange`
  向在线玩家增量广播某个玩家状态。
- `BroadcastAnnouncement`
  广播系统公告。
- `BuildPlayerInfo`
  统一构造大厅玩家摘要对象。

### 1.6 `DefaultGameFlow/Client/Modules/ClientUserModule.cs`

文件职责：

- 客户端用户流程模块。

关键处理函数：

- `OnS2C_LoginResult`
  登录成功后写入 `ClientSession`，并决定是否进入重连链。
- `OnS2C_ReconnectResult`
  恢复房间成功时创建 `ClientRoom` 并装配组件，随后发送 `C2S_ReconnectReady`。
- `OnS2C_KickOut`
  被顶号/踢下线时离房、清理会话并广播本地提示。

### 1.7 `DefaultGameFlow/Client/Modules/ClientRoomModule.cs`

## 2. RoomFlow

这一组脚本负责“房间成员基础流程”，与 `DefaultGameFlow` 的“房间生命周期”配合使用。

### 2.1 `RoomFlow/Server/Components/ServerRoomSettingsComponent.cs`

文件职责：

- 服务端房间基础设置组件。
- 负责房主、成员准备状态、成员快照、开局/结束控制。

核心字段：

- `_app`
  当前 `ServerApp`。
- `_readyStates`
  `SessionId -> IsReady`。
- `_ownerSessionId`
  当前房主。

关键函数：

- `OnInit`
  初始化房主和 ready 状态缓存。
- `CreateMemberInfo`
  组装 `MemberInfo`。
- `OnMemberJoined`
  维护房主、ready 状态并广播加入事件。
- `OnMemberLeft`
  删除 ready 状态并必要时迁移房主。
- `OnMemberOffline`
  房主离线时触发房主迁移。
- `MigrateHost`
  在剩余成员中优先选择在线成员成为房主。
- `OnSendSnapshot`
  给指定成员补发完整房间快照；如果房间已经开局，还会补 `S2C_GameStarted`。
- `BroadcastSnapshotToAll`
  向全体广播完整房间快照。
- `OnC2S_SetReady`
  更新成员 ready 状态并广播。
- `OnC2S_StartGame`
  仅房主且所有其他成员 ready 时允许开局。
- `OnC2S_EndGame`
  仅房主且当前在 `Playing` 态时允许强制结束。

### 2.2 `RoomFlow/Client/Components/ClientRoomSettingsComponent.cs`

文件职责：

- 客户端房间基础信息缓存组件。

核心字段/属性：

- `_app`
  所属 `ClientApp`。
- `Members`
  房间成员缓存表。
- `IsGameStarted`
  当前是否已开局。
- `RoomName`
  房间名。
- `MaxMembers`
  最大人数。
- `IsPrivate`
  是否私有房。

关键函数：

- `OnInit`
  重置本地缓存。
- `OnDestroy`
  清空成员表。
- `OnS2C_RoomSnapshot`
  覆盖整个本地房间快照。
- `OnS2C_MemberJoined`
  增量加入成员。
- `OnS2C_MemberLeft`
  增量移除成员。
- `OnS2C_MemberReadyChanged`
  更新成员 ready 状态。
- `OnS2C_GameStarted`
  标记开局。
- `OnS2C_GameEnded`
  标记结束并重置全部 ready 状态。

## 3. NetworkMonitoring

这一组脚本负责 Ping、RTT、弱网告警、阻断与主动熔断。

### 3.1 `NetworkMonitoring/Runtime/NetworkMonitoringRuntimeFeature.cs`

文件职责：

- 弱网监控运行时桥。

成员：

- `_networkMonitor`
  当前挂在 `StellarNetAppManager` 上的 `ClientNetworkMonitor`。
- `OnClientAppCreated`
  注册 `C2S_Ping` 弱网豁免，创建并初始化监控器。
- `OnClientPacketReceived`
  任意收包都刷新监控器活跃时间。
- `OnClientStopped`
  清空监控器引用。

### 3.2 `NetworkMonitoring/Client/Infrastructure/ClientNetworkMonitor.cs`

文件职责：

- 客户端弱网监控器。

核心字段：

- `_app`
  当前 `ClientApp`。
- `_transport`
  当前物理传输层。
- `_isWarnTriggered`
  是否进入弱网告警态。
- `_isBlockTriggered`
  是否进入弱网阻断态。
- `_blockDuration`
  已连续阻断时长。
- `_uiRefreshTimer`
  UI 事件节流计时器。
- `_pingTimer`
  主动发 Ping 节流计时器。
- `_currentRttMs`
  当前 RTT。
- `_lastPongReceiveTime`
  最近收到 Pong 或任意网络包的时间。
- `WeakNetWarnRttMs`
  进入告警态阈值。
- `WeakNetBlockRttMs`
  进入阻断态阈值。
- `ActiveFuseTimeoutSeconds`
  阻断持续太久时主动断线阈值。

关键函数：

- `Init`
  绑定 `ClientApp` 与 transport，订阅 `Local_PingResult`。
- `OnPingResult`
  用 Pong 回来的时间戳计算 RTT。
- `OnPacketReceived`
  任意收包都刷新活动时间。
- `Update`
  每帧执行监控逻辑：
  - 非在线房间态时重置状态
  - 周期性发送 `C2S_Ping`
  - 超过阈值进入 warn/block
  - 阻断持续过久则主动 `_transport.StopClient()`
  - 广播 `Local_NetworkQualityChanged`
  - 更新 `ClientApp.IsNetworkBlocked`

### 3.3 `NetworkMonitoring/Client/Modules/ClientPingModule.cs`

文件职责：

- 把 `S2C_Pong` 转成本地 RTT 事件。

成员：

- `_app`
  当前 `ClientApp`。
- `OnS2C_Pong`
  通过 `Time.realtimeSinceStartup - msg.ClientTime` 计算 RTT，并广播 `Local_PingResult`。

### 3.4 `NetworkMonitoring/Server/Modules/ServerPingModule.cs`

文件职责：

- 服务端心跳回包模块。

成员：

- `_app`
  当前 `ServerApp`。
- `OnC2S_Ping`
  原样回送 `S2C_Pong.ClientTime`。

### 3.5 `NetworkMonitoring/Shared/Protocol/Module/PingProtocols.cs`

类型与字段：

- `C2S_Ping`
  - `ClientTime`
- `S2C_Pong`
  - `ClientTime`

### 3.6 `NetworkMonitoring/Client/Events/NetworkMonitoringLocalEvents.cs`

本地事件：

- `Local_NetworkQualityChanged`
  - `RttMs`
  - `IsWeakNetWarn`
  - `IsWeakNetBlock`
- `Local_PingResult`

## 4. ObjectSync

这一组脚本负责网络对象的创建、销毁、变换同步、动画同步、回放快照导入导出。

### 4.1 `ObjectSync/Runtime/ObjectSyncRuntimeFeature.cs`

文件职责：

- ObjectSync 运行时桥。

成员：

- `OnRuntimeAwake`
  预加载 `ObjectSync` 配置。

### 4.2 `ObjectSync/Shared/Infrastructure/ObjectSyncConfig.cs`

类型与字段：

- `ObjectSyncGlobalConfig`
  - `ReplayObjectSyncRecordIntervalTicks`
    回放记录对象同步快照的 tick 间隔。
  - `ObjectSyncOnlineIntervalTicks`
    在线同步间隔。
  - `EnableAdaptiveObjectSync`
    是否按房间人数启用自适应同步频率。
  - `ObjectSyncFullResyncIntervalTicks`
    全量重同步间隔。
- `ObjectSyncConfigLoader`
  - `ConfigFileName`
  - `_cachedConfig`
  - `Current`
  - `LoadRuntimeConfigSync/Async`

### 4.3 `ObjectSync/Shared/Protocol/Components/ObjectSync/ObjectSpawnState.cs`

文件职责：

- 单个网络对象出生快照。
- 实现 `ILiteNetSerializable`，属于高频/大体积二进制结构。

字段：

- `NetId`
- `PrefabHash`
- `Mask`
- `PosX/PosY/PosZ`
- `RotX/RotY/RotZ`
- `DirX/DirY/DirZ`
- `ScaleX/ScaleY/ScaleZ`
- `AnimStateHash`
- `AnimNormalizedTime`
- `AnimParamCount`
- `AnimParams`
- `OwnerSessionId`

### 4.4 `ObjectSync/Shared/Protocol/Components/ObjectSync/ObjectSyncProtocols.cs`

文件职责：

- 定义对象同步脏位、状态结构和消息协议。

核心类型：

- `EntitySyncMask`
  描述对象具备哪些同步能力，如 transform/animator/owner 等。
- `ObjectSyncDirtyMask`
  描述本帧哪些字段发生变化。
- `ObjectSyncState`
  单帧对象同步增量，字段包括：
  - `NetId`
  - `Mask`
  - `DirtyMask`
  - `PosX/PosY/PosZ`
  - `RotX/RotY/RotZ`
  - `VelX/VelY/VelZ`
  - `ScaleX/ScaleY/ScaleZ`
  - `AnimStateHash`
  - `AnimNormalizedTime`
  - `AnimParamCount`
  - `AnimParams`
- `S2C_ObjectSpawn`
  出生消息，内部持有 `ObjectSpawnState State`，并提供一组只读转发表达属性。
- 其余同文件协议通常包括：
  - `S2C_ObjectDestroy`
  - `S2C_ObjectSync`
  - 与回放快照/重同步相关的同步载体

这个文件是对象同步协议层核心，和 `ServerObjectSyncComponent` / `ClientObjectSyncComponent` 强绑定。

### 4.5 `ObjectSync/Server/Components/ServerObjectSyncComponent.cs`

文件职责：

- 服务端对象同步核心组件。

核心类型与字段：

- `ServerSyncEntity`
  描述单个服务端权威实体：
  - `NetId`
  - `PrefabHash`
  - `Mask`
  - `OwnerSessionId`
  - `Position`
  - `Rotation`
  - `Velocity`
  - `Scale`
  - `AnimStateHash`
  - `AnimNormalizedTime`
  - `AnimParams`
  - `AnimParamCount`
- `SyncSnapshot`
  上次发送基线，用于脏位计算。
- `ServerObjectSyncComponent`
  - `_app`
    当前 `ServerApp`
  - `_entities`
    权威实体表
  - `_lastSentSnapshots`
    上次发送基线表
  - `_netIdCounter`
    新实体 ID 自增器
  - 房间规模阈值：
    `MediumRoomMemberThreshold`、`LargeRoomMemberThreshold`
  - 动态同步间隔：
    `MediumRoomIntervalTicks`、`LargeRoomIntervalTicks`
  - 脏位阈值：
    `PositionDirtyThresholdSqr`、`RotationDirtyThreshold`、`VelocityDirtyThresholdSqr`、`ScaleDirtyThresholdSqr`、`FloatParamDirtyThreshold`、`AnimNormalizedTimeDirtyThreshold`
  - `_syncStateBuffer`
    复用的同步状态缓存
  - `_reusableSyncMsg`
    复用的同步消息对象
  - `SnapshotComponentId`
    该组件在回放快照中的组件 ID，当前为 `200`

职责拆解：

- 生成/销毁网络实体
- 维护权威 transform/velocity/animator 状态
- 按阈值算脏位
- 周期性广播增量或全量同步
- 导出对象同步回放快照

- `_unknownSyncedParamHashes`
- `_receivedAnimatorParamHashes`

这一版 `NetAnimatorView` 已经不是旧的“固定 3 个 float 参数 + 字符串状态列表”方案，而是完整升级为：

1. `SyncedStates`
   可视化的状态映射列表，每项包含：
   - `Enabled`
   - `LogicStateName`
   - `AnimatorStateName`
2. `SyncedFloatParams`
   可视化的动态 float 参数映射列表，每项包含：
   - `Enabled`
   - `LogicParamName`
   - `AnimatorParamName`
3. Inspector 自动扫描
   从 `Animator Controller` 自动列出所有状态和 float 参数。
4. Inspector 校验
   会直接提示状态不存在、参数不存在、重复逻辑名、反滑步列表填错等问题。
5. 运行时动态参数恢复
   如果服务端本帧没有继续发送某个已映射 float 参数，客户端会把该参数平滑恢复到 Animator 初始值，避免旧参数残留造成 BlendTree 误判。

动画同步的核心运行时职责如下：

1. `HardSetInitialState(...)`
   实体首次生成时，立即应用初始动画状态和参数。
2. `ProcessAnimatorSync(...)`
   每帧读取 `ClientObjectSyncComponent` 提供的预测动画数据。
3. `ResolveAnimatorStateHash(...)`
   把服务端逻辑状态 hash 映射成 Animator 真正可播放的状态 hash。
4. `ApplyAnimatorParams(...)`
   把服务端动态参数数组映射到 Animator float 参数，并进行平滑。
5. `ResetMissingAnimatorParams(...)`
   把本帧未收到的已映射参数回收到默认值。
6. `ShouldBlockTransitionIntoStopState(...)`
   反滑步判断，避免实体还在移动时就切进 `Idle`。

## 6. 这一层的阅读建议

如果你要快速掌握默认业务主链，建议顺序是：

1. `DefaultGameFlow/Server/Modules/ServerUserModule.cs`
2. `DefaultGameFlow/Server/Modules/ServerRoomModule.cs`
3. `DefaultGameFlow/Client/Modules/ClientUserModule.cs`
4. `DefaultGameFlow/Client/Modules/ClientRoomModule.cs`
5. `RoomFlow/Server/Components/ServerRoomSettingsComponent.cs`
6. `RoomFlow/Client/Components/ClientRoomSettingsComponent.cs`

如果你要查同步、回放或高负载问题，建议顺序是：

1. `ServerObjectSyncComponent.cs`
2. `ClientObjectSyncComponent.cs`
3. `ReplayRoomRecordingService.cs`
4. `ServerReplayStorage.cs`
5. `ClientReplayPlayer.cs`

如果你要查弱网或连接降级问题，建议顺序是：

1. `ClientNetworkMonitor.cs`
2. `ClientPingModule.cs`
3. `NetworkMonitoringRuntimeFeature.cs`
