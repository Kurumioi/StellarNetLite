# 12. StellarNet Lite 首次运行与 SocialDemo 跑通指南

> 本文不是框架开发指南，而是首跑指南。
> 目标只有一个：第一次接触仓库的人，按这篇文档就能把 `Samples/SocialDemo` 跑起来，并知道当前示例场景到底挂了什么、该怎么启动客户端和服务端。

---

## 1. 先记住三个结论

1. `Samples/SocialDemo` 已经是可直接运行的样例，不需要你第一次上手就自己新建场景、自己手动配完整启动链。
2. `StellarNetLiteGenerated/` 是自动生成目录，不要手改；真正需要阅读和维护的是 `StellarNetLite/`。
3. 当前仓库原本有“开发说明”和“快速开始”，但缺一篇把**场景挂载、启动顺序、双开方式、Demo 操作步骤**串起来的首跑教程，本文就是补这条链。

---

## 2. 这三个 KCP 场景分别是干什么的

当前建议优先使用：

- `Samples/SocialDemo/Client/Scenes/KCP/KCPClientScene`
- `Samples/SocialDemo/Client/Scenes/KCP/KCPHostScene`
- `Samples/SocialDemo/Client/Scenes/KCP/KCPServerScene`

它们对应的用途是：

- `KCPClientScene`：纯客户端
- `KCPHostScene`：本地 Host，同进程内同时启动服务端和客户端
- `KCPServerScene`：独立服务端 / 无头服务端入口

---

## 3. 当前示例场景里到底挂了什么

### 3.1 `KCPClientScene`

这个场景里最关键的两个对象是：

- `NetLauncher`
- `NetRoot`

挂载关系是：

- `NetLauncher`
  - `GameLauncher`
  - `netMode = Client`
  - `appManager` 指向 `NetRoot` 上的 `StellarNetAppManager`
- `NetRoot`
  - `StellarNetAppManager`
  - `KcpTransportProvider`

也就是说，**客户端场景已经配好**，按 Play 就会走：

`GameLauncher -> 读取 NetConfig -> ApplyConfig -> StartClient`

### 3.2 `KCPHostScene`

挂载关系和客户端场景一样，只是：

- `GameLauncher.netMode = Host`

按 Play 后会走：

`GameLauncher -> 读取 NetConfig -> ApplyConfig -> StartHost`

这也是**单机先验证主链最省事**的场景。

### 3.3 `KCPServerScene`

这个场景也有：

- `NetLauncher`
- `NetRoot`

挂载关系是：

- `NetLauncher`
  - `GameLauncher`
  - `netMode = Server`
- `NetRoot`
  - `StellarNetAppManager`
  - `KcpTransportProvider`
- 额外还有一个独立对象：
  - `HeadlessServerLogger`

按 Play 后会走：

`GameLauncher -> 读取 NetConfig -> ApplyConfig -> StartServer`

如果以后你要做独立服务端或无头服务端，这个场景就是现成入口。

### 3.4 一个容易误解的点

`Panel_StellarNetBoot` 已经不是当前 Demo 的启动入口。

当前样例改成了**直接用 `GameLauncher` 自动启动**，所以不要再按“先打开启动面板再选模式”的旧思路理解当前代码。

另外，`StellarNetAppManager` 所在根节点现在会在 `Awake()` 后自动 `DontDestroyOnLoad`。

这意味着：

- 客户端切场景时不会把网络根对象一起销毁
- 传输层组件因为和 `StellarNetAppManager` 挂在同一个对象上，也会一起跨场景保留
- 如果场景里又放了第二套网络根，运行时会自动销毁后来的重复实例

## 4. 第一次跑之前要准备什么

### 4.1 导入目录

导入 Unity 工程时，通常对应为：

- `Assets/StellarNetLite/`
- `Assets/StellarNetLiteGenerated/`

其中：

- `StellarNetLite/`：手写框架代码、示例、文档
- `StellarNetLiteGenerated/`：扫描器和生成器输出，不手改

### 4.2 依赖

当前工程需要：

- `Newtonsoft.Json`

### 4.3 先生成静态表

首次导入，或者你改过协议 / 模块 / 房间组件 / 网络预制体后，都要重新生成：

- `StellarNetLite/重新生成协议与组件常量表`
- `StellarNetLite/生成网络预制体常量表 (Net Prefabs)`

如果怀疑表脏了，再用：

- `StellarNetLite/强制重新生成协议与组件常量表`

### 4.4 配置文件

当前启动链默认从这里读配置：

- 先读取 `StreamingAssets/NetConfig/netconfig_root.json`
- 再根据里面记录的根目录，去读取真正的 `netconfig.json`

编辑器工具入口是：

- `StellarNetLite/网络配置 (NetConfig)`

当前真正的配置文件候选位置有两个：

- `StreamingAssets/NetConfig/netconfig.json`
- `PersistentDataPath/NetConfig/netconfig.json`

也就是说：

- 你在 EditorWindow 里把根目录切到 `PersistentDataPath` 后，运行时会跟着改成优先从沙盒目录读
- 再切回 `StreamingAssets` 后，运行时也会跟着切回资源目录读

需要注意的是：

- 根目录切换本身会立即更新引导文件
- 如果目标根目录里还没有 `netconfig.json`，运行时会回退默认配置
- `PersistentDataPath` 是当前设备自己的沙盒目录；你在 Editor 里保存到这里的配置，不会自动跟着 Build 带到另一台机器

如果你本地**还没有配置文件**，当前代码会回退默认配置，默认值是：

- `Ip = 127.0.0.1`
- `Port = 7777`
- `TickRate = 60`

所以第一次本地首跑，其实就算没手动建配置文件，也可以先按默认值验证。

## 5. 最简单的三种启动方式

### 5.1 只想最快先看到完整链路

直接打开：

- `KCPHostScene`

然后按 Play。

这个模式会在一个进程里同时启动：

- 服务端
- 本地客户端

适合先验证：

- 是否能自动启动
- 是否能连上本地服务端
- 是否能登录
- 是否能建房
- 是否能开局并进入社交房间

### 5.2 想跑“独立服务端 + 独立客户端”

先开：

- `KCPServerScene`

再开：

- `KCPClientScene`

但要注意一个现实限制：

- **同一个 Unity Editor 实例不能同时 Play 两个场景**

所以你需要二选一：

- 第二个 Unity Editor 实例
- 一个 Editor + 一个 Build 出来的客户端
- 一个独立服务端 Build + 一个客户端 Editor / Build

### 5.3 想本地双开两个玩家

最容易理解的方式是：

1. 一个实例跑 `KCPServerScene`
2. 两个实例都跑 `KCPClientScene`
3. 两个客户端用不同账号登录

另一种方式是：

1. 一个实例跑 `KCPHostScene`
2. 另一个实例跑 `KCPClientScene`

这样也能验证多人联机。

## 6. 进了 Demo 以后该怎么操作

### 6.1 客户端启动后的第一屏

客户端连接建立后，`GameLauncher` 会初始化：

- `GlobalUIRouter`
- `RoomViewManager`
- `Panel_GlobalNetMonitor`
- `Panel_StellarNetLogin`

所以你在客户端场景里按 Play 后，正常应该看到：

- 网络监控面板
- 登录面板

### 6.2 登录

在 `Panel_StellarNetLogin`：

1. 输入账号
2. 点击登录

如果服务端没开，或者物理连接断了，登录面板里还会显示重连入口。

### 6.3 大厅

登录成功后会进入 `Panel_StellarNetLobby`。

这里能做的事情包括：

- 查看房间列表
- 创建房间
- 查看在线玩家
- 大厅聊天
- 查看录像列表

### 6.4 建房

点击“创建房间”后会打开 `Panel_SetRoomConfig`。

你需要填：

- 房间名
- 最大人数
- 可选的自定义 `RoomId`
- 房间模板

当前 Demo 默认房间模板来自：

- `Samples/SocialDemo/Shared/Binders/RoomTypeTemplateRegistry.cs`

默认模板内挂的是：

- `SocialRoom`
- `RoomSettings`
- `ObjectSync`

这意味着当前示例的房间能力默认就包含：

- 基础房间成员和准备态
- 社交房玩法组件
- 实体同步

### 6.5 进房与开局

进入房间后会打开 `Panel_StellarNetRoom`。

这里的流程是：

1. 所有人进入房间
2. 非房主点击 Ready
3. 房主点击 Start Game

当前服务端规则是：

- 只有房主能开始游戏
- 房主之外的成员都准备后，才能真正开局

如果房间里只有房主自己一个人，房主可以直接开始。

### 6.6 进入社交房间后的操作

开局后会打开 `Panel_SocialRoomView`。

当前默认操作是：

- `WASD` / 方向键：移动角色
- `1`：动作 1
- `2`：动作 2
- 输入框 + 发送按钮 / 回车：发送头顶气泡
- `结束游戏` 按钮：房主结束游戏
- `F12`：房主快速结束游戏

结束后会回到结算流程，随后可以：

- 离开房间
- 给录像改名
- 回到大厅查看录像列表
- 下载并进入回放

## 7. 为什么很多 UI 和 View 没直接挂在场景里

这是当前 Demo 一个很容易让新人误判的点。

你打开场景时，可能会发现：

- 登录 UI 没预摆一堆面板
- 房间里的表现层对象也没全摆在场景里

原因是现在很多东西是**运行时动态装配**的。

例如：

- 客户端连上服务端后，`GameLauncher` 才会打开登录和监控面板
- 进入房间后，`RoomViewManager` 才会创建 `[View] Room_xxx`
- 如果房间带 `ClientObjectSyncComponent`，会动态挂 `ObjectSpawnerView`
- 如果房间带 `ClientRoomSettingsComponent`，会动态挂对应的房间 UI Router
- 如果房间带 `ClientSocialRoomComponent`，会动态挂 `SocialRoomInputController` 和社交房 UI Router

所以：

- **首跑时不要误以为“场景里没挂全，所以还没配完”**
- 对 `SocialDemo` 来说，很多表现层就是运行时拉起来的

## 8. 如果你要自己新建一个启动场景，要挂什么

当你不再使用现成的 `KCPClientScene / KCPHostScene / KCPServerScene`，而是自己搭场景时，最小启动链是：

### 8.1 网络根对象

在同一个 GameObject 上挂：

- `StellarNetAppManager`
- 一种传输层实现

当前仓库内置可选：

- `KcpTransportProvider`
- `TcpTransportProvider`
- `UdpTransportProvider`

### 8.2 启动器对象

另一个对象上挂：

- `GameLauncher`

并把它的：

- `appManager`

指向上面那个 `StellarNetAppManager`。

然后根据用途设置：

- `netMode = Client`
- `netMode = Server`
- `netMode = Host`

### 8.3 无头服务端

如果你要服务端控制台、日志保留、命令输入，再额外挂：

- `HeadlessServerLogger`

## 9. 首跑最常见的坑

### 9.1 打开了 `KCPServerScene`，却在等登录 UI

这是正常的。

`KCPServerScene` 是**纯服务端入口**，不是客户端场景。

### 9.2 服务端和客户端都想在同一个 Editor 里分别 Play

做不到。

一个 Unity Editor 实例一次只能有一个 Play Session。

要么：

- 用 `KCPHostScene`

要么：

- 第二个 Editor / Build

### 9.3 改了协议或房间组件，运行却没反应

大概率是忘了重新生成静态表。

先重新跑：

- `重新生成协议与组件常量表`
- `生成网络预制体常量表`

### 9.4 本地没有 `netconfig.json`

当前不会直接卡死。

代码会回退默认配置：

- `127.0.0.1:7777`

但如果你用了多个实例，还是要确认它们最终读到的是同一套地址和端口。

### 9.5 你写了新房间组件，但进房后没效果

这通常不是启动问题，而是模板问题。

你除了写组件本身，还要把组件 Id 挂进房间模板，例如：

- `RoomTypeTemplateRegistry`

### 9.6 手改了 `StellarNetLiteGenerated`

不建议这样做。

这个目录应该始终视为生成产物。

## 10. 建议的首跑顺序

如果你现在的目标只是“先把 Demo 跑通”，建议按这个顺序：

1. 先看本文
2. 直接打开 `KCPHostScene` 跑一次单机主链
3. 再尝试 `KCPServerScene + KCPClientScene`
4. 然后回头看 `Doc/01` 和 `Doc/02`，理解启动链和传输层
5. 最后再看 `Doc/11`，进入业务开发练习
