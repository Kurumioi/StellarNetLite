# 05. StellarNet Lite 回放重连与框架自定义指南

> **核心导读**：这是本系列教程的高阶篇。我们将揭开 StellarNet Lite 最强大的两个高级特性：**流式沙盒回放**与**断线重连恢复**的神秘面纱。
> 同时，我们将教你如何进行深度的框架自定义：彻底抛弃默认的 3D 实体同步组件，利用快照接口从零编写一套属于你自己的同步系统（如 2D 同步），并让它完美兼容框架的回放与重连功能。

## 1. 回放系统为什么能做到“中段跳转 (Seek)”？

在很多简陋的网络框架中，回放就是“把历史收到的网络包，从第 0 秒开始重新播一遍”。
这种做法有一个致命缺陷：**如果你想快进到第 10 分钟，系统必须在后台把前 10 分钟的包疯狂运算一遍，极度卡顿，且容易出错。**

### StellarNet Lite 的解法：对象关键帧 (Snapshot)
StellarNet Lite 的录像文件（`.replay`）里，不仅存了普通的“消息帧”（比如谁发了表情，谁扣了血），还定期（比如每 150 帧）存了一个 **对象关键帧（ReplayObjectSnapshotFrame）**。

**当你拖动进度条 Seek 到第 5 分钟时：**
1. 播放器（`ClientReplayPlayer`）会在录像文件里，往回找离第 5 分钟最近的一个“关键帧”。
2. 播放器把当前场景里的实体全部清空。
3. 播放器根据关键帧，把第 5 分钟时的实体瞬间全部实例化出来，摆到正确的位置。
4. 然后再从关键帧开始，补播几秒钟的普通消息帧，直到精确对齐你拖动的时间点。

**这就是为什么 StellarNet Lite 的回放跳转如此丝滑且稳定的底层机密！**

## 2. 断线重连是怎么无缝恢复的？

手机玩游戏，进电梯断网了，出来后怎么恢复？

### 2.1 软挂起 (ConnectionSuspended)
当底层网络库检测到断开时，`ClientApp` 不会立刻把玩家踢回登录界面，而是进入一个特殊状态：`ClientAppState.ConnectionSuspended`。此时，UI 路由会弹出一个半透明的遮罩：“正在尝试恢复连接...”。

### 2.2 顶号与快照定向下发
1. 客户端在后台疯狂重试连接。
2. 连上后，客户端重新发送 `C2S_Login`（带上之前的 AccountId）。
3. 服务端 `ServerUserModule` 发现：“咦，这个账号之前在房间里啊！” 于是把旧的僵尸连接踢掉，把新连接绑定到旧的 `Session` 上。
4. 服务端返回 `HasReconnectRoom = true`。
5. 客户端收到后，发送 `C2S_ConfirmReconnect` 和 `C2S_ReconnectReady`。
6. **最关键的一步**：服务端收到 Ready 后，会遍历房间里所有的 `RoomComponent`，触发它们的快照导出，把房间当前的所有状态一次性发给这个重连的玩家！

## 3. 核心基石：快照接口 (Snapshot Interfaces)

为了让你自己写的组件也能享受上述的“录像跳转”和“断线重连”黑科技，框架提供了两个极其重要的接口：

### 3.1 服务端：IReplaySnapshotProvider
当房间开始录像、定期打关键帧，或者玩家断线重连时，引擎会调用实现了该接口的组件的 `ExportSnapshot()` 方法。
你需要把当前组件内的所有核心状态序列化成一个 `byte[]` 字节数组交出去。

```csharp
public interface IReplaySnapshotProvider
{
    int SnapshotComponentId { get; } // 必须与客户端的 Consumer ID 严格一致
    byte[] ExportSnapshot();         // 将当前状态序列化为字节流
}
```

### 3.2 客户端：IReplaySnapshotConsumer
当客户端看录像拖动进度条，或者重连进房收到快照包时，引擎会把刚才服务端生成的 `byte[]` 字节数组，精准地喂给实现了该接口的客户端组件。
你需要在这里反序列化字节流，并**瞬间重建**本地的表现层画面。

```csharp
public interface IReplaySnapshotConsumer
{
    int SnapshotComponentId { get; }
    void ApplySnapshot(byte[] payload); // 接收字节流，瞬间重建本地状态
}
```

## 4. 实战：完全抛弃默认同步，手写自定义 2D 实体同步

假设我们正在开发一款 2D 俯视角射击游戏，不需要复杂的 3D 旋转和 Animator 同步。我们将彻底抛弃底层的 `ObjectSyncComponent`，从零手写一套极简的 2D 同步组件。

### 第一步：定义自定义同步协议 (Shared)
```csharp
using System.IO;
using StellarNet.Lite.Shared.Core;
using StellarNet.Lite.Shared.Infrastructure;

namespace MyGame.Shared.Protocol
{
    // 1. 定义极简的 2D 状态结构
    public struct Custom2DSyncState : ILiteNetSerializable
    {
        public int NetId;
        public float PosX;
        public float PosY;
        public float FacingAngle;

        public void Serialize(BinaryWriter writer)
        {
            writer.Write(NetId);
            writer.Write(PosX);
            writer.Write(PosY);
            writer.Write(FacingAngle);
        }

        public void Deserialize(BinaryReader reader)
        {
            NetId = reader.ReadInt32();
            PosX = reader.ReadSingle();
            PosY = reader.ReadSingle();
            FacingAngle = reader.ReadSingle();
        }
    }

    // 2. 定义高频同步包
    [NetMsg(8001, NetScope.Room, NetDir.S2C)]
    public sealed class S2C_Custom2DSync : ILiteNetSerializable
    {
        public int ValidCount;
        public Custom2DSyncState[] States;

        public void Serialize(BinaryWriter writer)
        {
            writer.Write(ValidCount);
            for (int i = 0; i < ValidCount; i++) States[i].Serialize(writer);
        }

        public void Deserialize(BinaryReader reader)
        {
            ValidCount = reader.ReadInt32();
            if (States == null || States.Length < ValidCount) States = new Custom2DSyncState[ValidCount];
            for (int i = 0; i < ValidCount; i++) States[i].Deserialize(reader);
        }
    }
}
```

### 第二步：编写服务端自定义同步组件 (Server)
继承 `ServerRoomComponent`，实现 `ITickableComponent`（用于高频广播）和 `IReplaySnapshotProvider`（用于支持录像与重连）。

```csharp
using System.Collections.Generic;
using System.IO;
using StellarNet.Lite.Server.Core;
using StellarNet.Lite.Shared.Core;
using StellarNet.Lite.Shared.Replay;
using MyGame.Shared.Protocol;

namespace MyGame.Server.Components
{
    [RoomComponent(800, "Custom2DSync", "自定义2D实体同步")]
    public sealed class ServerCustom2DSyncComponent : ServerRoomComponent, ITickableComponent, IReplaySnapshotProvider
    {
        private readonly Dictionary<int, Custom2DSyncState> _entities = new Dictionary<int, Custom2DSyncState>();
        private Custom2DSyncState[] _syncBuffer = new Custom2DSyncState[64];
        private readonly S2C_Custom2DSync _syncMsg = new S2C_Custom2DSync();

        // 快照 ID，必须与客户端一致
        public int SnapshotComponentId => 800;

        public override void OnInit()
        {
            _entities.Clear();
            _syncMsg.States = _syncBuffer;
        }

        // 核心：高频心跳驱动同步
        public void OnTick()
        {
            if (Room.State != RoomState.Playing || _entities.Count == 0) return;
            
            // 假设每 3 帧同步一次
            if (Room.CurrentTick % 3 != 0) return;

            int index = 0;
            foreach (var kvp in _entities)
            {
                _syncBuffer[index++] = kvp.Value;
            }
            _syncMsg.ValidCount = index;
            
            // 广播给所有客户端，并写入录像文件
            Room.BroadcastMessage(_syncMsg, true);
        }

        // 核心：实现快照导出，支持录像 Seek 和断线重连
        public byte[] ExportSnapshot()
        {
            if (_entities.Count == 0) return System.Array.Empty<byte>();

            using (MemoryStream ms = new MemoryStream())
            using (BinaryWriter writer = new BinaryWriter(ms))
            {
                writer.Write(_entities.Count);
                foreach (var kvp in _entities)
                {
                    kvp.Value.Serialize(writer);
                }
                return ms.ToArray();
            }
        }
    }
}
```

### 第三步：编写客户端自定义同步组件 (Client)
继承 `ClientRoomComponent`，实现 `IReplaySnapshotConsumer`。接收服务端的同步包与快照，并抛出本地事件驱动 View 层刷新。

```csharp
using System.Collections.Generic;
using System.IO;
using StellarNet.Lite.Client.Core;
using StellarNet.Lite.Shared.Core;
using StellarNet.Lite.Shared.Replay;
using MyGame.Shared.Protocol;

namespace MyGame.Client.Components
{
    [RoomComponent(800, "Custom2DSync", "自定义2D实体同步")]
    public sealed class ClientCustom2DSyncComponent : ClientRoomComponent, IReplaySnapshotConsumer
    {
        public int SnapshotComponentId => 800;

        // 缓存本地状态供表现层查询
        private readonly Dictionary<int, Custom2DSyncState> _localEntities = new Dictionary<int, Custom2DSyncState>();

        public override void OnInit()
        {
            _localEntities.Clear();
        }

        // 接收常规高频同步包
        [NetHandler]
        public void OnS2C_Custom2DSync(S2C_Custom2DSync msg)
        {
            if (msg == null || msg.States == null) return;

            for (int i = 0; i < msg.ValidCount; i++)
            {
                var state = msg.States[i];
                _localEntities[state.NetId] = state;
            }
            
            // 抛出事件，通知表现层（如自建的 2DSpawnerView）更新位置
            Room.NetEventSystem.Broadcast(msg);
        }

        // 核心：处理录像 Seek 或断线重连时的瞬间状态恢复
        public void ApplySnapshot(byte[] payload)
        {
            if (payload == null || payload.Length == 0)
            {
                _localEntities.Clear();
                return;
            }

            using (MemoryStream ms = new MemoryStream(payload))
            using (BinaryReader reader = new BinaryReader(ms))
            {
                int count = reader.ReadInt32();
                _localEntities.Clear();

                for (int i = 0; i < count; i++)
                {
                    var state = new Custom2DSyncState();
                    state.Deserialize(reader);
                    _localEntities[state.NetId] = state;
                }
            }

            // 抛出快照重建事件，通知 View 层销毁旧实体，瞬间生成新实体
            Room.NetEventSystem.Broadcast(new Local_Custom2DSnapshotApplied());
        }
    }

    // 本地事件定义
    public struct Local_Custom2DSnapshotApplied { }
}
```

### 第四步：替换房间装配模板
最后，在 `RoomTypeTemplateRegistry.cs` 中，将默认的 `ComponentIdConst.ObjectSync` 移除，换成你刚才定义的 `800`。

```csharp
new RoomTypeTemplate
{
    TypeName = "我的2D定制房间",
    ComponentIds = new[]
    {
        ComponentIdConst.RoomSettings,
        800 // 替换为你的 Custom2DSync 组件 ID
    }
}
```

### 总结
通过以上四步，你已经完全抛弃了框架底层的 3D 实体同步逻辑，建立了一套极简的 2D 同步系统。
最重要的是，由于你正确实现了 `IReplaySnapshotProvider` 和 `IReplaySnapshotConsumer`，**你的这套自定义同步系统将完美、无缝地支持 StellarNet Lite 强大的录像快进与断线重连功能！** 这就是 MSV 架构与组件化设计的终极魅力。
