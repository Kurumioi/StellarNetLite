# 04. StellarNet Lite 实体同步与动画同步深度解析

本文档旨在深入剖析 StellarNet Lite 框架中的实体空间同步（Transform）与动画同步（Animator）机制。面向多人游戏开发新手，本文将从底层架构心智出发，详细说明“为什么这样设计”、“如何规范使用”，并提供两种常见同步模型（客户端权威与服务端权威）的完整实战案例，最后给出工程排查与模块拓展指南。

---

## 一、 核心架构与设计理念

在传统的高级网络库中，开发者通常依赖 `[SyncVar]` 或 `NetworkTransform` 等组件实现自动同步。这种“黑盒”机制在原型开发阶段较为便利，但在商业化项目中极易引发带宽失控、广播风暴以及难以追溯的脏数据问题。

StellarNet Lite 遵循**“显式数据流”**与**“MSV 架构解耦”**的设计哲学：

### 1. 显式状态快照 (Explicit State Snapshot)
- **服务端 (Model/Service)**：`ServerObjectSyncComponent` 维护当前房间内所有活跃实体的状态（位置、速度、动画 Hash 等）。它按固定的 TickRate（如每秒 20 次），将这些状态打包为紧凑的二进制数组（`S2C_ObjectSync`），显式广播给客户端。
- **客户端 (Model)**：`ClientObjectSyncComponent` 接收到同步包后，**不直接操作场景物体**，而是将数据写入本地的预测缓冲区。
- **表现层 (View)**：挂载在 GameObject 上的 `NetTransformView` 和 `NetAnimatorView` 在 `Update` 生命周期中读取缓冲区数据，驱动表现层刷新。

### 2. 空间同步原理：Dead Reckoning 与 Soft Reconciliation
网络传输必然存在延迟与抖动。如果直接将接收到的坐标赋值给 `transform.position`，会导致画面严重卡顿与瞬移。
`NetTransformView` 采用以下策略抹平网络抖动：
- **阈值判定**：计算当前坐标与服务端最新坐标的距离。如果距离过大（如 > 3米），判定为严重滞后或瞬移，直接拉扯（Snap）到目标点。
- **平滑追赶**：如果距离在合理范围内，利用服务端下发的当前速度（Velocity），结合 `Vector3.SmoothDamp` 算法，平滑地向预测点逼近。

### 3. 动画同步原理：Anti-Ice Skating (防滑冰机制)
动画同步的常见痛点是“滑冰”（角色模型在平移，但播放的是站立动画，或反之）。
`NetAnimatorView` 同步了 `AnimStateHash`（状态机短名 Hash）、`AnimNormalizedTime`（播放进度）与 `FloatParams`（混合树参数）。
- **防滑冰拦截**：当服务端下发 `Idle`（站立）状态，但客户端表现层检测到该实体由于网络插值仍在向前滑动时，视图层会**暂缓**切入 `Idle` 状态，直到实体物理速度真正降为 0 后再播放，从而彻底消除滑冰视觉表现。

## 二、 基础使用规范

将一个普通 GameObject 接入 StellarNet Lite 同步体系，需遵循以下标准流程：

1. **预制体装配**：
    - 在根节点挂载 `NetIdentity` 组件。
    - 挂载 `NetTransformView`（处理位置与旋转）。
    - （可选）挂载 `NetAnimatorView` 并绑定 Animator（处理动画）。
    - 将预制体存入 `Resources` 目录。
2. **生成常量表**：
    - 点击顶部菜单 `StellarNetLite -> 生成网络预制体常量表`。工具会扫描并生成 `NetPrefabConsts` 静态类。
3. **客户端沙盒装配**：
    - 确保客户端场景中存在挂载了 `ObjectSpawnerView` 的节点，并在进入房间时调用 `Init(room)`。该组件负责监听实体生成/销毁事件，并自动实例化对应的预制体。

## 三、 实战案例 1：状态转发模式 (客户端权威)

**适用场景**：社交大厅、虚拟展厅、对防作弊要求不高的合作类游戏。
**核心逻辑**：客户端本地直接计算物理移动（绝对跟手，零延迟），将计算后的坐标与速度定期上报给服务端；服务端仅做简单的合法性校验（防瞬移），随后覆写实体状态并广播给其他客户端。

### 1. 协议定义
```csharp
using StellarNet.Lite.Shared.Core;
using StellarNet.Lite.Shared.Infrastructure;
using System.IO;

namespace StellarNet.Lite.Game.Shared.Protocol
{
    [NetMsg(1301, NetScope.Room, NetDir.C2S)]
    public sealed class C2S_SocialMoveReq : ILiteNetSerializable
    {
        public float PosX, PosY, PosZ;
        public float VelX, VelY, VelZ;
        public float RotY;

        public void Serialize(BinaryWriter writer)
        {
            writer.Write(PosX); writer.Write(PosY); writer.Write(PosZ);
            writer.Write(VelX); writer.Write(VelY); writer.Write(VelZ);
            writer.Write(RotY);
        }
        public void Deserialize(BinaryReader reader)
        {
            PosX = reader.ReadSingle(); PosY = reader.ReadSingle(); PosZ = reader.ReadSingle();
            VelX = reader.ReadSingle(); VelY = reader.ReadSingle(); VelZ = reader.ReadSingle();
            RotY = reader.ReadSingle();
        }
    }
}
```

### 2. 服务端校验与转发
```csharp
using StellarNet.Lite.Game.Shared.Protocol;
using StellarNet.Lite.Server.Components;
using StellarNet.Lite.Server.Core;
using StellarNet.Lite.Shared.Core;
using UnityEngine;
using System.Collections.Generic;

namespace StellarNet.Lite.Game.Server.Components
{
    [RoomComponent(102, "SocialRoom", "社交房间")]
    public sealed class ServerSocialRoomComponent : RoomComponent
    {
        private ServerObjectSyncComponent _syncService;
        private readonly Dictionary<string, int> _sessionToNetId = new Dictionary<string, int>();
        private static readonly int Hash_Idle = Animator.StringToHash("Idle");
        private static readonly int Hash_Walk = Animator.StringToHash("Walk");

        public override void OnInit()
        {
            _syncService = Room.GetComponent<ServerObjectSyncComponent>();
        }

        public override void OnMemberJoined(Session session)
        {
            if (Room.State == RoomState.Playing) SpawnPlayer(session);
        }

        private void SpawnPlayer(Session session)
        {
            if (_syncService == null) return;
            var entity = _syncService.SpawnObject(
                StellarNet.Lite.Shared.Protocol.NetPrefabConsts.Prefab_SocialPlayer, 
                EntitySyncMask.All, 
                new Vector3(0, 0, 0), Vector3.zero, Vector3.zero, session.SessionId);
            
            if (entity != null)
            {
                entity.AnimStateHash = Hash_Idle;
                _sessionToNetId[session.SessionId] = entity.NetId;
            }
        }

        [NetHandler]
        public void OnC2S_SocialMoveReq(Session session, C2S_SocialMoveReq msg)
        {
            if (!_sessionToNetId.TryGetValue(session.SessionId, out int netId)) return;
            var entity = _syncService.GetEntity(netId);
            if (entity == null) return;

            Vector3 newPos = new Vector3(msg.PosX, msg.PosY, msg.PosZ);
            Vector3 newVel = new Vector3(msg.VelX, msg.VelY, msg.VelZ);

            // 简单防作弊校验：距离突变拦截
            float distance = Vector3.Distance(entity.Position, newPos);
            if (distance > 10.0f && entity.Position != Vector3.zero) 
            {
                // 判定为异常瞬移，拒绝覆写
                return; 
            }

            // 覆写服务端实体状态
            entity.Position = newPos;
            entity.Velocity = newVel;
            entity.Rotation = new Vector3(0, msg.RotY, 0);

            // 根据速度推导动画状态
            entity.AnimStateHash = newVel.sqrMagnitude > 0.01f ? Hash_Walk : Hash_Idle;
        }
    }
}
```

### 3. 客户端本地驱动与上报
```csharp
using StellarNet.Lite.Client.Core;
using StellarNet.Lite.Game.Shared.Protocol;
using UnityEngine;

namespace StellarNet.Lite.Game.Client.Views
{
    public class SocialLocalPlayerController : MonoBehaviour
    {
        private CharacterController _cc;
        private float _lastSendTime;
        private const float SendInterval = 0.05f; // 20Hz
        private Vector2 _lastInput;

        private void Awake()
        {
            _cc = GetComponent<CharacterController>();
        }

        private void Update()
        {
            float h = Input.GetAxisRaw("Horizontal");
            float v = Input.GetAxisRaw("Vertical");
            Vector3 inputDir = new Vector3(h, 0, v).normalized;
            
            // 本地直接移动，保证零延迟手感
            _cc.Move(inputDir * 4.0f * Time.deltaTime);
            
            if (inputDir.sqrMagnitude > 0.1f)
            {
                transform.rotation = Quaternion.LookRotation(inputDir);
            }

            Vector2 currentInput = new Vector2(h, v);
            bool inputChanged = currentInput != _lastInput;
            bool timeToSync = Time.realtimeSinceStartup - _lastSendTime >= SendInterval;

            // 节流上报：输入改变或持续移动时按频率上报
            if (inputChanged || (timeToSync && currentInput.sqrMagnitude > 0))
            {
                _lastInput = currentInput;
                _lastSendTime = Time.realtimeSinceStartup;
                
                NetClient.Send(new C2S_SocialMoveReq
                {
                    PosX = transform.position.x,
                    PosY = transform.position.y,
                    PosZ = transform.position.z,
                    VelX = _cc.velocity.x,
                    VelY = _cc.velocity.y,
                    VelZ = _cc.velocity.z,
                    RotY = transform.eulerAngles.y
                });
            }
        }
    }
}
```

## 四、 实战案例 2：指令驱动模式 (服务端权威)

**适用场景**：竞技类 FPS、生存游戏、强对抗场景。
**核心逻辑**：客户端仅上报纯粹的输入指令（如虚拟摇杆方向、是否开火）；服务端集中进行物理推演与碰撞检测，随后将计算出的权威坐标下发给所有客户端。

### 1. 协议定义
```csharp
using StellarNet.Lite.Shared.Core;
using StellarNet.Lite.Shared.Infrastructure;
using System.IO;

namespace StellarNet.Lite.Game.Shared.Protocol
{
    [NetMsg(2001, NetScope.Room, NetDir.C2S)]
    public sealed class C2S_FPSInputCmd : ILiteNetSerializable
    {
        public float InputX, InputZ;
        public float RotY;
        public bool IsFire;

        public void Serialize(BinaryWriter writer)
        {
            writer.Write(InputX); writer.Write(InputZ);
            writer.Write(RotY); writer.Write(IsFire);
        }
        public void Deserialize(BinaryReader reader)
        {
            InputX = reader.ReadSingle(); InputZ = reader.ReadSingle();
            RotY = reader.ReadSingle(); IsFire = reader.ReadBoolean();
        }
    }
}
```

### 2. 服务端物理推演
```csharp
using StellarNet.Lite.Game.Shared.Protocol;
using StellarNet.Lite.Server.Components;
using StellarNet.Lite.Server.Core;
using StellarNet.Lite.Shared.Core;
using UnityEngine;
using System.Collections.Generic;

namespace StellarNet.Lite.Game.Server.Components
{
    [RoomComponent(501, "FPSRoom", "竞技FPS房间")]
    public sealed class ServerFPSRoomComponent : RoomComponent, ITickableComponent
    {
        private ServerObjectSyncComponent _syncService;
        private readonly Dictionary<string, int> _playerNetIds = new Dictionary<string, int>();
        private static readonly int Hash_Idle = Animator.StringToHash("Idle");
        private static readonly int Hash_Run = Animator.StringToHash("Run");
        private static readonly int Hash_Fire = Animator.StringToHash("Fire");

        public override void OnInit()
        {
            _syncService = Room.GetComponent<ServerObjectSyncComponent>();
        }

        // ... SpawnPlayer 逻辑与案例 1 类似 ...

        [NetHandler]
        public void OnC2S_FPSInputCmd(Session session, C2S_FPSInputCmd msg)
        {
            if (!_playerNetIds.TryGetValue(session.SessionId, out int netId)) return;
            var entity = _syncService.GetEntity(netId);
            if (entity == null) return;

            // 记录客户端输入，转化为期望速度
            Vector3 inputDir = new Vector3(msg.InputX, 0, msg.InputZ).normalized;
            entity.Velocity = inputDir * 5.0f; // 基础移速 5m/s
            entity.Rotation = new Vector3(0, msg.RotY, 0);

            if (msg.IsFire)
            {
                entity.AnimStateHash = Hash_Fire;
                entity.AnimNormalizedTime = 0f; // 强制重置动画进度
            }
            else
            {
                entity.AnimStateHash = entity.Velocity.sqrMagnitude > 0.1f ? Hash_Run : Hash_Idle;
            }
        }

        public void OnTick()
        {
            // 服务端权威推演：根据当前 Velocity 更新 Position
            // 在实际项目中，这里应当调用服务端的物理引擎 (如 Physics.Simulate) 进行碰撞检测
            float dt = 1f / Room.Config.TickRate;
            foreach (var netId in _playerNetIds.Values)
            {
                var entity = _syncService.GetEntity(netId);
                if (entity != null && entity.Velocity.sqrMagnitude > 0)
                {
                    entity.Position += entity.Velocity * dt;
                }
            }
        }
    }
}
```

### 3. 客户端指令上报
```csharp
using StellarNet.Lite.Client.Core;
using StellarNet.Lite.Game.Shared.Protocol;
using UnityEngine;

namespace StellarNet.Lite.Game.Client.Views
{
    public class FPSInputReporter : MonoBehaviour
    {
        private float _lastSendTime;
        private const float SendInterval = 0.05f;

        private void Update()
        {
            if (NetClient.State != ClientAppState.OnlineRoom) return;

            float h = Input.GetAxisRaw("Horizontal");
            float v = Input.GetAxisRaw("Vertical");
            bool isFire = Input.GetMouseButtonDown(0);
            
            // 节流上报
            if (isFire || Time.realtimeSinceStartup - _lastSendTime >= SendInterval)
            {
                _lastSendTime = Time.realtimeSinceStartup;
                NetClient.Send(new C2S_FPSInputCmd 
                { 
                    InputX = h, 
                    InputZ = v, 
                    RotY = transform.eulerAngles.y,
                    IsFire = isFire
                });
            }
        }
    }
}
```

## 五、 业务模块横向拓展指南

自带的 `ObjectSyncComponent` 仅负责 Transform 与 Animator 的高频同步。在实际项目中，实体通常还包含血量(HP)、体力、背包等数据。

**核心原则：严禁修改底层的 `ObjectSyncState` 结构体。**
StellarNet Lite 提倡通过**横向新增 Component** 来解耦业务。

### 拓展示例：同步玩家血量
1. **定义协议**：新建 `S2C_HealthSync` 协议，包含 `NetId` 和 `CurrentHP`。
2. **服务端组件 (`ServerHealthComponent`)**：
   维护一个 `Dictionary<int, int>` 记录血量。当玩家受到伤害时，修改字典，并调用 `Room.BroadcastMessage(new S2C_HealthSync {...})`。
   *注：血量变化属于低频事件，仅在数值改变时单次广播即可，切勿放入 `OnTick` 中每帧发送。*
3. **客户端组件 (`ClientHealthComponent`)**：
   通过 `[NetHandler]` 接收 `S2C_HealthSync`，随后利用 `Room.NetEventSystem.Broadcast(msg)` 将事件抛出给表现层。
4. **表现层 (View)**：
   UI 脚本或头顶血条组件监听该本地事件，并刷新进度条。

## 六、 常见工程排查指南

- **Q1: 为什么我的物体在客户端没有生成？**
    - 检查预制体是否位于 `Resources` 目录。
    - 检查是否执行了常量表生成工具。
    - 检查客户端场景中是否挂载了 `ObjectSpawnerView` 并成功调用了 `Init(room)`。

- **Q2: 为什么远端玩家移动时一卡一卡的？**
    - 检查服务端是否按预期频率（如 20Hz）更新并下发了实体的 `Position` 与 `Velocity`。
    - 检查客户端 `NetTransformView` 的 `PosSmoothTime` 设置。若值过小（如 0.01），平滑效果会失效；若值过大（如 0.5），会导致表现严重滞后。默认值 `0.1` 适用于大多数 20Hz 同步场景。

- **Q3: 为什么动画状态没有同步？**
    - `NetAnimatorView` 依赖 Animator 的**状态短名 (State Name)** 进行同步，而非 Transition Trigger。确保服务端赋值的 `Animator.StringToHash("Walk")` 中，"Walk" 是 Animator 状态机里真实存在的 State 节点名称。
