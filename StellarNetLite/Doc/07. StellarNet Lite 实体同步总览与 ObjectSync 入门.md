## 1. `ObjectSync` 解决什么问题

在多人房间项目里，服务端需要把实体状态同步给多个客户端。

这里的“实体状态”通常至少包含两部分：

1. 空间状态
   - 位置
   - 旋转
   - 速度
   - 缩放
2. 动画状态
   - 当前逻辑动画状态
   - 动画归一化时间
   - 动态 float 参数

`ObjectSync` 就是用来解决这件事的。

它的设计目标有四个：

1. 服务端权威
   什么状态最终生效，以服务端写入的数据为准。
2. 客户端平滑表现
   客户端收到数据后不是硬跳，而是做预测、平滑、过渡。
3. 回放友好
   同步数据能进入快照和重放流程。
4. 表现层解耦
   网络层不直接耦合你的具体角色脚本和 UI。

## 2. 这套系统有哪些核心组成

### 2.1 服务端核心

- `ServerObjectSyncComponent`
  负责服务端实体注册、脏检查、生成同步包、导出快照。
- 业务房间组件
  例如 `ServerSocialRoomComponent`，负责决定某个实体什么时候移动、什么时候播动画。

### 2.2 客户端核心

- `ClientObjectSyncComponent`
  接收服务端同步消息，缓存实体状态，提供预测后的 Transform / Animator 数据。
- `ObjectSpawnerView`
  收到生成消息后，负责把网络实体实例化到场景中。
- `NetIdentity`
  保存 `NetId` 和 `SyncService`，让视图组件知道自己对应哪一个网络实体。
- `NetTransformView`
  把同步出来的空间数据应用到 Unity `Transform`。
- `NetAnimatorView`
  把同步出来的动画状态和参数应用到 Unity `Animator`。

### 2.3 共享协议

- `ObjectSpawnState`
  实体首次生成时发送的完整状态。
- `ObjectSyncState`
  实体后续增量同步时发送的状态。
- `S2C_ObjectSpawn`
  生成实体消息。
- `S2C_ObjectDestroy`
  销毁实体消息。
- `S2C_ObjectSync`
  批量同步实体状态消息。

## 3. 一个网络实体从 0 跑起来的完整流程

这一节非常重要。

### 3.1 服务端生成实体

业务层通常会调用：

```csharp
ServerSyncEntity entity = _syncService.SpawnObject(
    prefabHash,
    EntitySyncMask.All,
    position,
    rotation,
    velocity,
    ownerSessionId);
```

这里会做几件事：

1. 创建一个 `ServerSyncEntity`
2. 分配唯一 `NetId`
3. 保存这个实体当前的 Transform / Animator 状态
4. 生成 `ObjectSpawnState`
5. 广播 `S2C_ObjectSpawn`

### 3.2 客户端收到生成消息

客户端的 `ClientObjectSyncComponent.OnS2C_ObjectSpawn(...)` 会先缓存生成状态。

然后 `ObjectSpawnerView` 会：

1. 根据 `PrefabHash` 找到对应 Prefab
2. 实例化这个 Prefab
3. 交给 `ObjectSyncClientRuntime.DefaultNetViewBinder` 做默认绑定

### 3.3 默认绑定会补什么组件

默认绑定器会尽量确保实例上有这些组件：

1. `NetIdentity`
2. `NetTransformView`
3. `NetAnimatorView`

然后：

1. `NetIdentity.Init(...)`
2. `NetTransformView.HardSetInitialState(...)`
3. `NetAnimatorView.HardSetInitialState(...)`

### 3.4 后续增量同步怎么走

服务端每个 Tick 会做：

1. 检查实体哪些字段真的变化了
2. 生成 `ObjectSyncDirtyMask`
3. 只把变化字段写进 `ObjectSyncState`
4. 打包到 `S2C_ObjectSync`
5. 批量发给客户端

客户端收到后：

1. `ClientObjectSyncComponent` 更新缓存
2. `NetTransformView` 每帧读取预测后的 Transform 数据
3. `NetAnimatorView` 每帧读取预测后的 Animator 数据
4. 把表现应用到场景对象

## 4. `EntitySyncMask` 是什么

`EntitySyncMask` 决定一个实体需要同步哪些大类数据。

定义在：

- `ObjectSyncProtocols.cs`

当前有三个值：

1. `EntitySyncMask.None`
   不同步任何内容
2. `EntitySyncMask.Transform`
   只同步空间数据
3. `EntitySyncMask.Animator`
   只同步动画数据
4. `EntitySyncMask.All`
   同时同步空间和动画

大多数可移动角色建议直接用：

```csharp
EntitySyncMask.All
```

## 5. `ObjectSpawnState` 和 `ObjectSyncState` 区别是什么

### 5.1 `ObjectSpawnState`

这是“完整状态”。

特点：

1. 新实体生成时使用
2. 字段完整
3. 客户端拿到后可以直接还原实体当前状态

里面既有：

- `PosX/PosY/PosZ`
- `RotX/RotY/RotZ`
- `DirX/DirY/DirZ`
- `ScaleX/ScaleY/ScaleZ`

也有：

- `AnimStateHash`
- `AnimNormalizedTime`
- `AnimParamCount`
- `AnimParams`

### 5.2 `ObjectSyncState`

这是“增量状态”。

特点：

1. 后续实时同步使用
2. 配合 `DirtyMask`
3. 只传本帧真正变化的字段

这样做的好处是：

1. 节省带宽
2. 减少无意义同步
3. 在高负载房间里更稳

## 6. 新手第一次做网络实体时该怎么操作

### 6.1 准备 Prefab

准备一个普通 Unity Prefab，例如：

- 玩家角色
- NPC
- 可移动物件

### 6.2 挂 `NetIdentity`

根节点挂：

- `NetIdentity`

### 6.3 如果要同步位移，挂 `NetTransformView`

它负责：

1. 位置平滑
2. 旋转平滑
3. 追赶
4. Snap
5. CharacterController 兼容

### 6.4 如果要同步动画，挂 `NetAnimatorView`

它负责：

1. 状态映射
2. 参数映射
3. 参数平滑
4. 反滑步
5. 回放恢复

### 6.5 把 Prefab 放到可被网络加载的位置

通常会放在：

- `Client/Resources/NetPrefabs/`

然后重新生成 `NetPrefabConsts.cs`。

### 6.6 服务端生成它

业务层通过：

```csharp
_syncService.SpawnObject(...)
```

把它生成到房间中。

## 7. 服务端业务层到底要不要手写同步逻辑

要。

但要分清楚手写的是什么。

### 7.1 服务端不直接操作 Unity `Animator`

服务端不会像客户端那样写：

- `animator.Play(...)`
- `animator.SetFloat(...)`

因为服务端这里本质上是在写“网络状态”，不是在写 Unity 场景表现。

### 7.2 服务端需要手写的是“逻辑状态”

例如：

1. 玩家速度大于 0
   - 切 `Walk`
2. 玩家停下
   - 切 `Idle`
3. 玩家按了挥手
   - 切 `Wave`

也就是说，服务端负责决定：

1. 当前逻辑状态是什么
2. 当前逻辑参数是多少

再把它们写入 `ServerSyncEntity`。

### 7.3 当前仓库里有现成案例

案例文件：

- `Samples/SocialDemo/Server/Components/ServerSocialRoomComponent.cs`

例如：

```csharp
syncEntity.SetAnimState("Idle");
```

移动时：

```csharp
if (playerSync.AnimStateHash != AnimHash_Walk)
{
    playerSync.SetAnimState("Walk");
}
else if (playerSync.AnimStateHash == AnimHash_Walk)
{
    playerSync.SetAnimState("Idle");
}
```

动作时：

```csharp
playerSync.SetAnimState("Wave");
playerSync.SetAnimState("Dance");
```

以后如果你要驱动 BlendTree，也是在服务端这样写：

```csharp
entity.SetAnimState("Move");
entity.SetAnimParam("MoveSpeed", speed);
entity.SetAnimParam("MoveX", moveX);
entity.SetAnimParam("MoveY", moveY);
```

## 9. 一句话记忆

你可以把 `ObjectSync` 理解成：

1. 服务端维护实体真实状态
2. 协议层负责把完整状态和增量状态发出去
3. 客户端缓存并预测这些状态
4. `NetTransformView` 和 `NetAnimatorView` 负责把它们表现成 Unity 场景中的可见结果
