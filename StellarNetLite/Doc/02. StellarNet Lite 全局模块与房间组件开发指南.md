# 02. StellarNet Lite 全局模块与房间组件开发指南

> **保姆级教学**：本篇将手把手带你从零开发一个“大厅全服广播”和一个“房间内发表情”的功能。学完本篇，你就能完全摆脱 Demo，开始写自己的游戏逻辑了。

---

## 1. 核心前置概念：Global 还是 Room？

在写代码前，先问自己一个问题：**这个功能是在大厅用的，还是在房间里用的？**

- 如果是**大厅聊天、查看排行榜、拉取邮件** -> 属于全局域（Global），需要写 **全局模块（Module）**。
- 如果是**战斗扣血、发房间表情、同步位置** -> 属于房间域（Room），需要写 **房间组件（RoomComponent）**。

---

## 2. 偷懒神器：业务脚手架生成器

框架为你准备了代码生成工具，不用手敲那些繁琐的类名和特性（Attribute）。

点击菜单：`StellarNetLite -> 业务脚手架生成器`

在弹出的窗口中：
1. **脚手架类型**：选择 `GlobalModule`（全局模块）或 `RoomComponent`（房间组件）。
2. **输出根目录**：选择你想存放脚本的文件夹（比如 `Assets/Scripts/Game`）。
3. **模块名 (英文)**：比如填入 `Broadcast` 或 `Emoji`。
4. **显示名 (中文)**：比如填入 `全服广播模块`。
5. **MsgId**：给协议分配一个唯一的 ID（如 800，801）。
6. 点击 **生成源码**。

框架会自动帮你生成 `Shared`、`Server`、`Client` 三个文件夹下的样板代码！接下来我们看看这些代码长什么样，以及如何填入真实逻辑。

---

## 3. 实战一：开发全局模块（大厅全服广播）

**目标**：玩家在大厅点击按钮，发送一条广播，所有在线玩家都能收到。

### 第一步：定义共享协议 (Shared)
打开生成的 `BroadcastProtocols.cs`，定义客户端发给服务端的请求（C2S），和服务端广播给客户端的同步（S2C）。

```csharp
using StellarNet.Lite.Shared.Core;

namespace MineGame.Shared.Protocol
{
    // C2S 表示 Client To Server，客户端发给服务端
    [NetMsg(800, NetScope.Global, NetDir.C2S)]
    public sealed class C2S_BroadcastReq
    {
        public string Content; // 玩家想说的话
    }

    // S2C 表示 Server To Client，服务端发给客户端
    [NetMsg(801, NetScope.Global, NetDir.S2C)]
    public sealed class S2C_BroadcastSync
    {
        public string SenderName; // 谁发的
        public string Content;    // 说了什么
    }
}
```

### 第二步：编写服务端权威逻辑 (Server)
打开生成的 `ServerBroadcastModule.cs`，处理客户端发来的请求。

```csharp
using StellarNet.Lite.Server.Core;
using StellarNet.Lite.Shared.Core;
using MineGame.Shared.Protocol;

namespace MineGame.Server.Modules
{
    [ServerModule("ServerBroadcastModule", "全服广播模块")]
    public sealed class ServerBroadcastModule
    {
        private readonly ServerApp _app;

        public ServerBroadcastModule(ServerApp app) { _app = app; }

        // 加上 [NetHandler] 特性，框架就会自动把 C2S_BroadcastReq 路由到这里
        [NetHandler]
        public void OnC2S_BroadcastReq(Session session, C2S_BroadcastReq msg)
        {
            // 1. 防御性编程：判空拦截
            if (session == null || string.IsNullOrWhiteSpace(msg.Content)) return;

            // 2. 组装要广播的 S2C 消息
            var syncMsg = new S2C_BroadcastSync
            {
                SenderName = session.Uid, // 从 Session 中获取玩家唯一标识
                Content = msg.Content
            };

            // 3. 遍历所有在线玩家，发送广播
            foreach (var kvp in _app.Sessions)
            {
                Session targetSession = kvp.Value;
                if (targetSession != null && targetSession.IsOnline)
                {
                    _app.SendMessageToSession(targetSession, syncMsg);
                }
            }
        }
    }
}
```

### 第三步：编写客户端接收逻辑 (Client)
打开生成的 `ClientBroadcastModule.cs`，接收服务端的广播，并抛出本地事件给 UI。

```csharp
using StellarNet.Lite.Client.Core;
using StellarNet.Lite.Client.Core.Events;
using StellarNet.Lite.Shared.Core;
using MineGame.Shared.Protocol;

namespace MineGame.Client.Modules
{
    [ClientModule("ClientBroadcastModule", "全服广播模块")]
    public sealed class ClientBroadcastModule
    {
        private readonly ClientApp _app;

        public ClientBroadcastModule(ClientApp app) { _app = app; }

        [NetHandler]
        public void OnS2C_BroadcastSync(S2C_BroadcastSync msg)
        {
            if (msg == null) return;

            // 客户端模块不直接操作 UI！而是把网络包作为事件抛出去
            GlobalTypeNetEvent.Broadcast(msg);
        }
    }
}
```
*(注：UI 怎么监听这个事件，我们将在下一篇《UI路由与表现层解耦指南》中详细讲解。)*

## 4. 实战二：开发房间业务组件（房间内发表情）

**目标**：玩家在房间内发表情，只有这个房间里的人能看到，并且支持断线重连时恢复表情总数。

### 第一步：定义协议 (Shared)
房间协议的 Scope 必须是 `NetScope.Room`。

```csharp
using StellarNet.Lite.Shared.Core;

namespace MineGame.Shared.Protocol
{
    [NetMsg(900, NetScope.Room, NetDir.C2S)]
    public sealed class C2S_SendEmojiReq { public int EmojiId; }

    [NetMsg(901, NetScope.Room, NetDir.S2C)]
    public sealed class S2C_EmojiSync 
    { 
        public string SenderSessionId; 
        public int EmojiId; 
    }

    // 用于断线重连或新玩家刚进房间时的“快照”恢复
    [NetMsg(902, NetScope.Room, NetDir.S2C)]
    public sealed class S2C_EmojiSnapshot { public int TotalCount; }
}
```

### 第二步：编写服务端组件 (Server)
服务端组件继承自 `RoomComponent`，它能直接访问 `Room` 对象。

```csharp
using StellarNet.Lite.Server.Core;
using StellarNet.Lite.Shared.Core;
using MineGame.Shared.Protocol;

namespace MineGame.Server.Components
{
    [RoomComponent(10, "RoomEmoji", "房间表情组件")]
    public sealed class ServerEmojiComponent : RoomComponent
    {
        private int _totalCount; // 记录房间内一共发了多少个表情

        public override void OnInit()
        {
            _totalCount = 0; // 组件初始化时清零
        }

        [NetHandler]
        public void OnC2S_SendEmojiReq(Session session, C2S_SendEmojiReq msg)
        {
            if (session == null || Room == null) return;

            _totalCount++; // 修改权威状态

            var syncMsg = new S2C_EmojiSync
            {
                SenderSessionId = session.SessionId,
                EmojiId = msg.EmojiId
            };

            // 房间内广播！只有这个房间的人能收到
            Room.BroadcastMessage(syncMsg);
        }

        // 核心机制：当有玩家断线重连，或者中途加入房间时，框架会自动调用这个方法
        public override void OnSendSnapshot(Session session)
        {
            Room.SendMessageTo(session, new S2C_EmojiSnapshot { TotalCount = _totalCount });
        }
    }
}
```

### 第三步：编写客户端组件 (Client)
客户端组件继承自 `ClientRoomComponent`。注意，它抛出事件使用的是 `Room.NetEventSystem`，而不是全局事件！

```csharp
using StellarNet.Lite.Client.Core;
using StellarNet.Lite.Shared.Core;
using MineGame.Shared.Protocol;

namespace MineGame.Client.Components
{
    [RoomComponent(10, "RoomEmoji", "房间表情组件")]
    public sealed class ClientEmojiComponent : ClientRoomComponent
    {
        public int LocalTotalCount { get; private set; } // 客户端的轻状态缓存

        public override void OnInit() { LocalTotalCount = 0; }

        [NetHandler]
        public void OnS2C_EmojiSync(S2C_EmojiSync msg)
        {
            if (msg == null || Room == null) return;
            
            LocalTotalCount++;
            // 抛出房间级事件！这样回放房间和在线房间就不会串线！
            Room.NetEventSystem.Broadcast(msg);
        }

        [NetHandler]
        public void OnS2C_EmojiSnapshot(S2C_EmojiSnapshot msg)
        {
            if (msg == null || Room == null) return;

            LocalTotalCount = msg.TotalCount; // 恢复快照状态
            Room.NetEventSystem.Broadcast(msg);
        }
    }
}
```

## 5. 极其重要的一步：生成静态装配表！

你写完了上面的代码，如果直接运行，**什么都不会发生，甚至会报错！**
因为框架的底层是**零反射**的，它根本不知道你写了这些类。

**必须执行**：
点击 Unity 顶部菜单：`StellarNetLite -> 强制重新生成协议与组件常量表`

此时，编辑器工具会扫描你的 `[NetMsg]`、`[ServerModule]`、`[RoomComponent]`，并在 `Assets/StellarNetLiteGenerated/` 目录下生成静态字典。
只有生成完毕后，你的协议才能被正确路由，你的房间组件才能在建房时被正确装配！

## 6. 如何在建房时带上你的新组件？

在客户端发起建房请求时，把你的组件 ID（刚才脚手架里填的 10）加进 `ComponentIds` 数组里即可：

```csharp
var createReq = new C2S_CreateRoom
{
    RoomName = "我的表情房",
    ComponentIds = new int[] 
    { 
        ComponentIdConst.RoomSettings, // 基础房间设置 (ID:1)
        10 // 你的 RoomEmoji 组件 ID
    }
};
NetClient.Send(createReq);
```
*(注：在正式项目中，建议将这些 ID 组合配置在 `RoomTypeTemplateRegistry.cs` 中，让 UI 通过选择“房间模板”来建房，而不是硬编码 ID。)*
