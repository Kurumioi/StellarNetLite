# 00. StellarNet Lite 文档总览与学习路径

本文档用于说明三件事：

1. 这个仓库的目录结构和各目录职责。
2. 核心脚本、扩展模块、示例工程之间的关系。
3. 新人从哪里开始看，怎样逐步建立完整理解。

---

## 1. 仓库定位

`StellarNet Lite` 是一套面向 Unity 房间制联机项目的 C# 网络框架。

如果你是第一次接触这个工程，先建立一个非常重要的目录认知：

- Unity 工程根目录是你打开 `StellarNetLite.sln`、`Packages/`、`ProjectSettings/` 的那一层目录。
- 框架源码主目录在 `Assets/StellarNetLite/`。
- 运行时配置目录在 `Assets/StreamingAssets/NetConfig/`。

很多新手第一次看文档会把：

- `Assets/StellarNetLite/...`

和：

- Unity 工程根目录下的 `StreamingAssets/...`

混在一起。

当前仓库实际结构是：

- 框架代码放在 `Assets/StellarNetLite/`
- 运行时配置放在 `Assets/StreamingAssets/`

先把这件事记住，后面看场景路径、配置路径、生成路径时就不会迷路。

当前仓库不是单纯的示例工程，而是由下面几部分共同组成：

- `Runtime`：框架底层运行时骨架。
- `Extensions`：默认功能扩展，例如登录、大厅、对象同步、回放、弱网监控。
- `Samples`：基于框架实现的示例业务，这里是 `SocialDemo`。
- `Editor`：代码生成、配置编辑、脚手架、监控和压测工具。
- `StreamingAssets`：运行时配置和独立压测工具。
- `StellarNetLiteGenerated`：编辑器扫描后生成的静态注册代码。

框架当前的设计原则如下：

- 服务端权威，客户端负责输入、缓存和表现。
- `Runtime` 只保留底层主链，不直接写死默认业务。
- 协议、模块、房间组件通过编辑器生成静态注册表接入运行时。
- 表现层与网络层解耦，业务逻辑尽量不写在 UI 中。

## 2. 根目录说明

仓库根目录当前主要包含以下内容：

- `README.md`
  说明项目定位、能力概览和快速开始方式。
- `LICENSE.text`
  项目许可协议。
- `StellarNetLite/`
  Unity 侧主要源码目录，包含 `Runtime`、`Extensions`、`Samples`、`Editor`、`Doc`。
- `StellarNetLiteGenerated/`
  编辑器扫描生成的静态绑定代码占位目录。
- `StreamingAssets/`
  运行时配置、压测工具、配置根目录引导文件所在目录。

## 3. 主目录职责

### 3.1 `StellarNetLite/Runtime`

框架运行时核心目录。

这里放的是所有项目都会依赖的底层骨架，不依赖具体业务。

关键子目录如下：

- `Client/Core`
  客户端主状态机、房间对象、房间工厂、客户端事件系统。
- `Server/Core`
  服务端主状态机、会话、房间、房间工厂、房间组件基类。
- `Server/Infrastructure`
  服务端运行时宿主、房间调度器、异步持久化运行时、无头日志控制台。
- `Shared/Core`
  协议元信息、传输接口、包结构、特性定义。
- `Shared/Infrastructure`
  配置加载、统一日志、PlayerLoop 主线程调度器。
- `Transports`
  `KCP`、`TCP`、`UDP` 三种传输层实现。

### 3.2 `StellarNetLite/Extensions`

官方默认扩展目录。

这里的代码不是底层必须能力，而是可插拔的默认实现。

当前主要扩展如下：

- `DefaultGameFlow`
  登录、大厅、建房、进房、挂起房间、重连确认等默认业务主链。
- `RoomFlow`
  房间成员、房主、准备状态、开局、结算等通用房间流程。
- `ObjectSync`
  网络实体创建、销毁、同步、预测、动画同步、快照导出与恢复。
- `Replay`
  房间录像录制、落盘、下载、客户端回放。
- `NetworkMonitoring`
  Ping、RTT 监控、弱网告警、发送阻断和主动熔断。

### 3.3 `StellarNetLite/Samples`

示例业务目录。

当前仓库提供的示例是 `SocialDemo`，用于演示以下内容：

- 登录和大厅流程。
- 房间配置和进房流程。
- 房间内成员、聊天、开局、结束。
- 社交房间中的对象同步和动画同步。
- 录像下载与回放。

### 3.4 `StellarNetLite/Editor`

Unity 编辑器工具目录。

这里的工具负责把框架从“源码”变成“可运行工程”。

主要脚本如下：

- `LiteProtocolScanner.cs`
  扫描 `[NetMsg]`、`[RoomComponent]`、`[ServerModule]`、`[ClientModule]`、`[NetHandler]`，生成静态绑定代码。
- `NetPrefabScanner.cs`
  扫描挂载 `NetIdentity` 的预制体，生成网络预制体常量表。
- `NetConfigEditorWindow.cs`
  统一编辑 Runtime、Replay、ObjectSync 三套配置。
- `StellarNetScaffoldWindow.cs`
  生成房间组件或全局模块的业务脚手架。
- `ServerMonitorWindow.cs`
  在编辑器中查看服务端运行时数据。
- `LoadTestWindow.cs`
  启动和控制独立压测工具。
- `FolderContentCopyTool.cs`
  用于复制模板或目录内容的辅助工具。

### 3.5 `StellarNetLite/Doc`

项目文档目录。

这里的文档按学习顺序组织，目标是让新手能从“跑起来”逐步过渡到“能改、能扩、能排查问题”。

### 3.6 `StellarNetLiteGenerated`

自动生成代码目录。

这个目录中的脚本由编辑器扫描器生成，运行时会直接依赖它们。

主要文件如下：

- `MsgIdConst.cs`
  协议消息 Id 常量表。
- `ComponentIdConst.cs`
  房间组件 Id 常量表。
- `AutoMessageMetaRegistry.cs`
  协议类型与元数据映射表。
- `AutoUnauthenticatedProtocolRegistry.cs`
  未鉴权允许访问的全局协议白名单。
- `AutoRegistry.cs`
  全局模块和房间组件的自动装配入口。
- `AutoRuntimeFeatureRegistry.cs`
  运行时扩展桥自动注册表。
- `NetPrefabConsts.cs`
  网络预制体 hash 常量表。

### 3.7 `StreamingAssets`

运行时配置和独立压测工具目录。

当前主要包含两部分：

- `NetConfig/`
  `netconfig.json`、`replay_config.json`、`objectsync_config.json` 以及运行时根目录引导文件。
- `StellarNetLiteLoadTest/`
  独立 .NET 压测工具源码、项目文件和启动脚本。

## 4. 关键脚本说明

下面这些脚本决定了框架的主链结构，建议优先理解。

### 4.1 运行时入口

- `Runtime/StellarNetAppManager.cs`
  整个框架的统一入口。负责装配传输层、序列化器、客户端宿主、服务端宿主和运行时扩展桥。

### 4.2 协议注册中心

- `Runtime/Shared/Core/NetMessageMapper.cs`
  运行时协议查询中心。只读自动生成表，不做运行时反射扫描。

### 4.3 客户端主状态机

- `Runtime/Client/Core/ClientApp.cs`
  客户端状态流转、消息发送、当前房间生命周期管理中心。

### 4.4 服务端主状态机

- `Runtime/Server/Core/ServerApp.cs`
  服务端消息收发、会话索引、房间索引、全局模块管理中心。

### 4.5 房间对象

- `Runtime/Server/Core/RoomInstance/Room.cs`
  服务端房间实例，负责成员管理、组件驱动、广播、录制和 Tick。
- `Runtime/Client/Core/ClientRoom.cs`
  客户端房间实例，负责房间组件、房间消息分发和房间事件总线。

### 4.6 房间组件与全局模块装配

- `Runtime/Server/Core/ServerRoomFactory.cs`
  服务端房间组件创建与绑定入口。
- `Runtime/Client/Core/ClientRoomFactory.cs`
  客户端房间组件创建与绑定入口。
- `StellarNetLiteGenerated/AutoRegistry.cs`
  自动装配代码聚合入口。

### 4.7 服务端线程模型

- `Runtime/Server/Infrastructure/ServerRuntimeHost.cs`
  服务端独立线程宿主，驱动网络泵、主 Tick 和出站队列。
- `Runtime/Server/Infrastructure/ServerRoomScheduler.cs`
  房间 worker 调度器，把房间分配到多个工作线程执行。

### 4.8 默认业务主链

- `Extensions/DefaultGameFlow/Server/Modules/ServerUserModule.cs`
  登录、顶号、重连确认、恢复流程。
- `Extensions/DefaultGameFlow/Server/Modules/ServerRoomModule.cs`
  建房、加房、进房最终确认、挂起房间、离房。
- `Extensions/DefaultGameFlow/Server/Modules/ServerLobbyModule.cs`
  房间列表、在线玩家列表、全局聊天、公告。

### 4.9 对象同步与回放

- `Extensions/ObjectSync/Server/Components/ServerObjectSyncComponent.cs`
  服务端对象同步核心。
- `Extensions/ObjectSync/Client/Components/ClientObjectSyncComponent.cs`
  客户端同步缓存、预测和回放快照消费核心。
- `Extensions/Replay/Server/Infrastructure/ReplayRoomRecordingService.cs`
  录像录制接入层。
- `Extensions/Replay/Client/Core/ClientReplayPlayer.cs`
  客户端回放播放器。

### 4.10 示例工程入口

- `Samples/SocialDemo/GameLauncher.cs`
  SocialDemo 启动入口，决定当前场景以 Client、Server 还是 Host 方式运行。
- `Samples/SocialDemo/Client/Infrastructure/GlobalUIRouter.cs`
  全局 UI 状态路由。
- `Samples/SocialDemo/Client/Infrastructure/RoomViewManager.cs`
  在线房间和回放房间的表现层装配总枢纽。

## 5. 模块关系

可以把当前仓库理解为下面这条链：

1. `Editor` 扫描协议、组件、模块和网络预制体。
2. 生成 `StellarNetLiteGenerated` 中的静态注册代码。
3. `StellarNetAppManager` 启动时加载这些静态表。
4. `ClientApp` / `ServerApp` 接管客户端和服务端主状态机。
5. `Extensions` 把默认业务链接入运行时。
6. `Samples/SocialDemo` 把业务层和表现层挂到默认链上。
7. `StreamingAssets/NetConfig` 提供运行时配置。
8. `StreamingAssets/StellarNetLiteLoadTest` 提供独立压测能力。

## 7. 建议先掌握的能力

如果当前目标是尽快上手业务开发，先掌握下面这些内容即可：

- 看懂 `ClientApp`、`ServerApp`、`Room`、`Session` 的职责边界。
- 分清什么逻辑该写在 `GlobalModule`，什么逻辑该写在 `RoomComponent`。
- 知道新增协议、模块、组件后要重新生成静态注册代码。
- 知道进房是两段式确认，不要在最终确认前发送房间域协议。
- 知道表现层要订阅事件，不要把网络主逻辑直接写进 UI。

## 8. 常见误区

- 以为 `Runtime` 已经包含登录、大厅、回放等完整业务逻辑。
  实际上这些能力主要在 `Extensions` 中。
- 以为新增协议后直接运行就能生效。
  实际上必须先重新生成自动注册代码。
- 以为客户端是逻辑真相来源。
  当前框架是服务端权威模型。
- 以为进房成功等于拿到 `CreateRoomResult` 或 `JoinRoomResult`。
  实际上最终还要等 `RoomSetupResult.Success == true`。
- 以为对象同步就是每帧全量广播。
  当前默认实现包含脏位同步、频率控制、全量校正和回放快照。
