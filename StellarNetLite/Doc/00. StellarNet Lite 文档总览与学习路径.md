# 00. StellarNet Lite 文档总览与学习路径

本文是整个文档体系的入口。

如果你第一次接触 `StellarNet Lite`，建议先看本文，再按推荐顺序往下读。这样你会更清楚：

1. 整个仓库哪些目录是核心
2. 文档各自负责讲什么
3. 新手应该先掌握哪些能力
4. 二开、排错、源码阅读应该从哪里切入

---

## 1. 仓库整体分层怎么理解

你可以先把整个仓库拆成几层：

### 1.1 `Runtime`

这里是最底层的运行时主链。

主要负责：

1. `ClientApp`
2. `ServerApp`
3. `Room`
4. `Session`
5. 协议分发
6. 传输层接入
7. Tick 调度

如果你想知道：

1. 客户端怎么连上服务端
2. 服务端怎么建房间
3. 协议怎么进入房间组件
4. Tick 怎么驱动业务

那最终都要回到 `Runtime`。

### 1.2 `Extensions`

这里是建立在 Runtime 之上的可复用扩展层。

当前最重要的几个扩展包括：

1. `DefaultGameFlow`
2. `RoomFlow`
3. `ObjectSync`
4. `Replay`
5. `NetworkMonitoring`

如果你在做实际项目，大多数“开箱即用能力”都会先在这里接入。

### 1.3 `Samples`

这里是样例工程。

当前最重要的是：

`SocialDemo`

它能帮助你快速看到：

1. 客户端如何启动
2. 服务端如何启动
3. 房间组件如何接业务
4. `ObjectSync` 如何用于真实实体

### 1.4 `Editor`

这里是编辑器工具。

主要负责：

1. 协议扫描与代码生成
2. NetPrefab 扫描
3. 配置编辑
4. 脚手架工具
5. 监控与压测工具

### 1.5 `Doc`

这里就是你现在在看的文档目录。

### 1.6 `StreamingAssets`

这里是运行时配置与一些外部工具文件。

例如：

1. `NetConfig/netconfig.json`
2. `NetConfig/replay_config.json`
3. `NetConfig/objectsync_config.json`

### 1.7 `StellarNetLiteGenerated`

这里是自动生成代码。

例如：

1. 协议 Id 常量
2. 组件 Id 常量
3. 自动注册表
4. Prefab Hash 映射

这个目录通常不要手改。

## 2. 新手最该先理解的几件事

如果你完全从 0 开始，先建立这几个认知：

1. 这是服务端权威框架，不是纯客户端主导框架。
2. 新增协议、组件、模块后，很多时候要重新生成自动注册代码。
3. `Runtime` 是主链，`Extensions` 是可复用扩展。
4. `Samples/SocialDemo` 是最好的第一手实战参考。
5. `ObjectSync`、`Replay`、`RoomFlow` 这些能力都是在 Runtime 之上接出来的。

## 4. 如果你是完全新手，最推荐的学习路径

### 第一步：先跑起来

先看：

1. `01`

目标不是理解全部原理，而是：

1. 把工程跑起来
2. 知道客户端和服务端如何启动
3. 看到一个真实可运行的样例

### 第二步：建立主链认知

再看：

1. `02`

目标是搞清楚：

1. 登录
2. 进房
3. 房间
4. 模块
5. 组件
6. 协议分发

这些词在框架里到底各指什么。

### 第三步：学会基本业务开发

再看：

1. `03`
2. `04`
3. `05`

目标是学会：

1. 新增一条协议
2. 新增一个模块
3. 新增一个房间组件
4. 知道什么时候要重新生成自动注册代码

### 第四步：进入同步实体开发

再看：

1. `07`
2. `07A`
3. `07B`

目标是：

1. 先理解 `ObjectSync` 全貌
2. 再掌握 `Transform` 同步
3. 再掌握 `Animator` 同步

这是当前新手最容易卡住，也是最需要细读的一组文档。

### 第五步：理解表现层、回放和重连

再看：

1. `08`
2. `09`

目标是：

1. 知道网络逻辑如何和 UI 解耦
2. 知道回放和断线重连如何接入

### 第六步：最后再做源码级深入

再看：

1. `14`
2. `15`
3. `16`
4. `17`

目标才是：

1. 源码级维护
2. 深度排错
3. 架构级二开

## 9. 当前最推荐优先掌握的能力

如果你现在只是想尽快开始做项目，建议优先掌握下面这些：

1. 看懂 `ClientApp`、`ServerApp`、`Room`、`Session` 的职责边界。
2. 分清什么逻辑该写在 `GlobalModule`，什么逻辑该写在 `RoomComponent`。
3. 知道新增协议、模块、组件后要重新生成自动注册代码。
4. 知道进房是两段式确认，不要在最终确认前发送房间域协议。
5. 知道表现层要订阅事件，而不要把网络主逻辑直接写进 UI。
6. 知道同步实体开发应优先看 `07 / 07A / 07B` 这一组文档。

## 10. 常见误区

### 误区一：以为 Runtime 已经包含完整业务

不是。

`Runtime` 更像底座。

登录、房间流程、对象同步、回放、监控这些更完整能力，大多在 `Extensions`。

### 误区二：以为改完协议直接运行就会生效

不是。

很多时候你还需要重新生成自动注册代码。

### 误区三：以为客户端是最终真相

不是。

当前整体路线是服务端权威。

### 误区四：以为同步角色只要看一个文档

不是。

现在推荐的理解顺序是：

1. `07` 看总览
2. `07A` 看 Transform
3. `07B` 看 Animator

这样比把所有内容塞进一篇更容易真正上手。
