# 00. StellarNet Lite 文档总览与学习路径

这份文档用于给初学者建立一条连续的学习顺序，并说明当前文档集合已经覆盖了框架的哪些部分。

当前仓库的目录分层已经调整为：

- `Runtime`：框架核心骨架
- `Extensions`：官方默认扩展能力
- `Samples`：示例业务与表现层
- `Editor`：编辑器工具
- `Doc`：教学文档

## 2. 为什么这样排序

### 第一阶段：先看主链

先读：

1. `01`
2. `02`
3. `03`

这一阶段解决的是：

- 框架怎么启动。
- 服务端和客户端怎么跑起来。
- KCP / TCP / UDP 怎么接到统一接口。
- 协议为什么会被分发到模块和房间组件。

### 第二阶段：再看房间运行和表现

再读：

4. `04`
5. `05`

这一阶段解决的是：

- 房间里的实体怎么同步。
- 动画怎么同步。
- 为什么 UI 不直接写进 Runtime。
- 在线房间和回放房间为什么要拆路由。

### 第三阶段：最后看高级能力、工具链和运维工具

最后读：

6. `06`
7. `07`
8. `08`
9. `09`
10. `10`

这一阶段解决的是：

- 回放为什么能 Seek。
- 重连为什么能恢复房间。
- 如何在尽量不改 Runtime 的前提下扩展业务。
- Editor 代码生成链如何参与自动装配。
- 无头服务端如何查看状态、日志和执行停机。
- 服务端异步持久化任务如何登记、观察和在 `exit` 时等待。

## 5. 旧版冗余文档怎么处理

之前独立存在的“注释规范”和“当前实现对齐说明”类型文档已经不再保留为单独教学文档。

现在的做法是：

- 注释规范直接落实到代码中。
- 实现对齐直接落实到各主题文档中。

这样更适合初学者顺着主线学习，也避免文档数量多但内容分散。

---

### 当前文档已额外覆盖的闭环细节

为避免“文档还是旧链路，代码已经收口”的问题，当前文档集已经额外覆盖这些关键点：

- 未鉴权会话默认只允许访问显式白名单 `Global` 协议；当前通过 `NetMsg.AllowUnauthenticated = true` 自动生成白名单，默认 `C2S_Login` 已声明放行
- 普通进房主链现在是 `授权进入 -> C2S_RoomSetupReady -> S2C_RoomSetupResult -> 正式进入`
- 重连主链现在是 `Login 恢复 -> ConfirmReconnect -> ReconnectReady -> S2C_RoomSetupResult -> Snapshot`
- 客户端在最终确认前会自动阻断 `NetScope.Room` 发包，业务层不需要自己补这层防线
- `ObjectSync` 的回放快照恢复现在会刷新已有实体状态

## 6. 想尽快开始写业务，建议这样读

如果你的目标不是先把底层全部吃透，而是尽快开始写自己的业务功能，推荐走这条更短的路径：

1. 先读 `12. StellarNet Lite 首次运行与 SocialDemo 跑通指南`
2. 再读 `01. StellarNet Lite 核心概念与全流程指南`
3. 再读 `03. StellarNet Lite 全局模块与房间组件开发指南`
4. 然后直接读 `11. StellarNet Lite 30分钟业务开发上手实践`
5. 真正动手新增协议、模块、组件时，再回看 `08. StellarNet Lite 代码生成与编辑器工具链指南`

按这条路径，开发者通常已经可以开始：

- 新增一个全局模块
- 新增一个房间业务组件
- 把房间组件挂进房间模板
- 重新生成自动装配代码
- 跑通最小业务链路

## 7. 初学者建议的实际上手顺序

如果你第一次接触这个仓库，建议按下面顺序动手：

1. 先看 `GameLauncher` 和 `StellarNetAppManager`。
2. 再看 `ServerRoomModule` 和 `ClientRoomModule`。
3. 然后看 `ServerRoomSettingsComponent` 和 `ClientRoomSettingsComponent`。
4. 再看 `ServerObjectSyncComponent` 和 `ClientObjectSyncComponent`。
5. 最后再看 `Samples/SocialDemo` 的 Router、Panel、View。

按这个顺序，理解成本最低。
