# 01. StellarNet Lite 核心概念与全流程指南

> **保姆级教学**：如果你是第一次接触 StellarNet Lite，请务必逐字阅读本文。我们将从最基础的心智建立开始，带你理清整个框架的运转脉络。

---

## 1. 核心心智：服务端才是唯一的真相

在单机游戏里，玩家按下“攻击键”，你可能会直接扣除怪物的血量并播放死亡动画。
但在 StellarNet Lite 中，**客户端绝对不能自己修改核心数据**。

正确的联机思维是：
1. **客户端（Client）**：玩家按下攻击键 -> 发送“我请求攻击怪物A”的网络包 -> 等待。
2. **服务端（Server）**：收到请求 -> 验证玩家是否存活、距离够不够、技能是否冷却 -> 扣除怪物血量 -> 广播“怪物A血量变为0，死亡”。
3. **客户端（Client）**：收到广播 -> 触发本地事件 -> UI 刷新血条，模型播放死亡动画。

**记住：客户端只发请求和播放结果，服务端负责计算真相。**

---

## 2. 框架的三层物理隔离

框架代码严格分为三个文件夹，它们有着不可逾越的界限：

- **Shared（共享层）**：
    - 存放双端都需要知道的东西：协议定义（发什么包）、常量表、数据结构。
    - **禁忌**：这里绝对不能写任何具体的业务逻辑或 UI 代码。
- **Server（服务端）**：
    - 存放权威逻辑：管理连接（Session）、管理房间（Room）、处理客户端请求。
    - **禁忌**：这里绝对不能出现 `UnityEngine.UI` 或任何跟画面表现相关的代码。
- **Client（客户端）**：
    - 存放表现逻辑：接收服务端广播、缓存状态、抛出本地事件驱动 UI 和模型。
    - **禁忌**：这里绝对不能直接修改核心业务数据（比如直接 `MonsterHp = 0`）。

---

## 3. 全局（Global）与房间（Room）作用域

框架将所有的网络行为分为两大类（Scope）：

- **Global（全局域）**：
    - 和具体的“对局”无关的行为。
    - 例如：登录账号、在大厅拉取房间列表、在大厅世界频道聊天、下载录像。
    - 负责处理这些的是 **全局模块（Module）**。
- **Room（房间域）**：
    - 必须在“进入房间”后才能发生的行为。
    - 例如：准备游戏、移动位置、攻击怪物、房间内发表情。
    - 负责处理这些的是 **房间组件（RoomComponent）**。

## 4. 数据是怎么流转的？（全流程拆解）

让我们以“玩家登录”为例，一步步看代码是怎么跑起来的。

### 第 1 步：发起请求（View 层）
玩家在 UI 面板输入账号，点击登录按钮。UI 脚本调用强类型发送接口：
```csharp
// 组装协议包
var msg = new C2S_Login { AccountId = "Player1", ClientVersion = "1.0.0" };
// 发送给服务端
NetClient.Send(msg);
```

### 第 2 步：客户端底层处理（ClientApp）
`NetClient.Send` 会将包交给 `ClientApp`。
`ClientApp` 会查阅**静态生成的协议表**，发现这是一个 `C2S`（Client to Server）方向的包，然后给它打上序号（Seq防重放），序列化成字节流，通过底层网络库（Mirror）发往服务器。

### 第 3 步：服务端接收与路由（ServerApp）
服务端的网络库收到字节流，交给 `ServerApp`。
`ServerApp` 解析出这是 `C2S_Login` 协议，发现它的作用域是 `Global`，于是把它交给全局分发器（GlobalDispatcher）。

### 第 4 步：服务端业务处理（Server Module）
分发器找到了负责处理登录的模块 `ServerUserModule`，调用它的处理函数：
```csharp
[NetHandler]
public void OnC2S_Login(Session session, C2S_Login msg)
{
    // 1. 验证版本号
    // 2. 绑定 Session
    // 3. 组装结果包
    var resultMsg = new S2C_LoginResult { Success = true, SessionId = session.SessionId };
    // 4. 发送回客户端
    _app.SendMessageToSession(session, resultMsg);
}
```

### 第 5 步：客户端接收与抛出事件（Client Module）
客户端底层收到 `S2C_LoginResult`，交给 `ClientUserModule`：
```csharp
[NetHandler]
public void OnS2C_LoginResult(S2C_LoginResult msg)
{
    if (msg.Success) {
        // 记录登录状态
        _app.Session.OnLoginSuccess(msg.SessionId, ...);
    }
    // 极其重要：将网络包作为事件抛出给表现层！
    GlobalTypeNetEvent.Broadcast(msg);
}
```

### 第 6 步：UI 刷新表现（View 层）
全局 UI 路由 `GlobalUIRouter` 早就监听了这个事件：
```csharp
GlobalTypeNetEvent.Register<S2C_LoginResult>(OnLoginResult);

private void OnLoginResult(S2C_LoginResult msg)
{
    if (msg.Success) {
        // 关闭登录面板，打开大厅面板
        UIKit.ClosePanel<Panel_StellarNetLogin>();
        UIKit.OpenPanel<Panel_StellarNetLobby>();
    }
}
```

**总结流转公式：**
`View 发包 -> Client 序列化 -> Server 验证并改状态 -> Server 下发结果 -> Client 抛出事件 -> View 监听事件并刷新 UI`

## 5. 房间是怎么建立和进入的？

StellarNet Lite 的房间不是一个写满逻辑的巨石类，而是**组件化（Component-based）**的。

### 5.1 建房的本质：点菜
客户端建房时，其实是在告诉服务端：“我要建一个房间，请给我装配 A、B、C 这几个功能组件。”
```csharp
var createReq = new C2S_CreateRoom
{
    RoomName = "新手村",
    ComponentIds = new int[] 
    { 
        ComponentIdConst.RoomSettings, // 基础设置组件
        ComponentIdConst.ObjectSync,   // 实体同步组件
        ComponentIdConst.Dungeon       // 你的自定义闯关组件
    }
};
NetClient.Send(createReq);
```

### 5.2 两阶段握手（极度重要）
进房并不是服务端直接把数据砸给客户端，而是有一个严谨的握手流程：

1. **服务端同意进房**：服务端验证通过，返回 `S2C_JoinRoomResult`，里面包含房间 ID 和组件清单。
2. **客户端本地建房**：客户端收到后，在本地创建一个 `ClientRoom`，并根据组件清单，把对应的 `ClientRoomComponent` 实例化并装配好。
3. **客户端就绪**：客户端本地装配完毕，发送 `C2S_RoomSetupReady` 告诉服务端：“我准备好了，可以发数据了”。
4. **服务端下发快照**：服务端收到就绪指令，正式把玩家加入房间，并触发所有组件的 `OnSendSnapshot`，把房间当前的所有状态（玩家列表、怪物位置等）一次性发给这个新玩家。

这样做的好处是：**绝对不会出现“客户端组件还没初始化，服务端的同步包就到了”的 Bug。**

## 6. 为什么一定要用 Editor 扫描生成？

在很多老旧框架中，收到网络包后，需要用 C# 的 `Reflection`（反射）去查找哪个函数该处理这个包。这在运行时非常消耗性能（产生大量 GC 和 CPU 开销）。

StellarNet Lite **完全零反射**。
当你写好一个新的协议或组件后，点击菜单：
`StellarNetLite -> 强制重新生成协议与组件常量表`

编辑器工具会扫描你的代码，自动生成类似这样的硬编码字典（静态装配表）：
```csharp
// 自动生成的代码，运行时直接 O(1) 查找，速度极快
target[100] = typeof(C2S_Login);
dispatcher.Register(100, (session, packet) => { ... });
```
**新手必坑点**：写了新协议发不出去？写了新组件进房报错？**99% 是因为你忘了点菜单重新生成！**
